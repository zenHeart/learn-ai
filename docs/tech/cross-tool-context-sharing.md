# 跨工具 Context 共享策略

## 核心要点

1. **Shared Context Bus 模式**：建立统一的 Context 中间层，各工具通过接口读写，而非各自维护独立 Context
2. **Context 序列化与反序列化**：工具间传递 Context 时需约定格式（JSON/Proto），确保结构化数据能正确解析
3. **生命周期管理**：Context 应有明确的创建、扩展、裁剪、销毁时机，避免内存泄漏
4. **按需加载**：工具按需从 Context Bus 获取所需片段，而非一次性加载全部，减少 Token 消耗
5. **冲突处理策略**：多工具可能产出冲突信息，需要优先级机制（如：最近优先/来源可信度优先）
