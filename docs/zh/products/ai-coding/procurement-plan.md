# 研发团队大模型（AI辅助编程）采购与实施方案

## 摘要（致 HR、财务及管理层）

为了让非技术背景的管理层和相关支持部门快速了解本次采购的核心价值及风险控制，特提炼以下核心要点：
1. **显著提升研发效能**：AI 辅助编程预计能提升研发团队 20%-30% 的编码与排错效率，帮助工程师减负，缩短项目交付周期。
2. **极高的资金利用率**：摒弃“按人头包月”或“买断硬件”的昂贵模式，采用“轻资产按量付费（Token 计费）+ 内部网关统筹”架构。10万元内预算足以支撑几十人团队一整年的高强度使用，且绝无资源闲置浪费。
3. **数据安全与合规可控**：通过内部网关拦截，员工无需接触原始 API 密钥，且能实现代码数据的脱敏过滤；优先使用国内备案合规的大模型，大幅降低使用海外模型的封停和数据出境风险。
4. **透明的人效与成本数据**：平台可精准统计每位员工、每个部门的 AI 工具调用量。这些数据可辅助 HR 和技术 Leader 评估工具的实际提效情况，并用于针对性的技术培训。

## 二、 成本预算、模型选择理由与实施方式

### 1. 实施方式
采用**“轻资产 SaaS API + 内部网关二次分发”**的混合云部署模式。
- **不采购**昂贵的 GPU 物理服务器。
- 直接调用云端 SaaS 大模型 API。
- 所有研发人员**不接触**原始 API Key。开发者的 IDE 插件（如 Cline、Continue 等）统一连接到公司内部部署的“统一大模型网关”。

### 2. 模型选择理由（高低搭配的混合路由策略）
考虑到成本效益和国内网络封号风险，我们采取不绑定单一厂商的策略：
- **80% 日常任务（主力部署：DeepSeek V3.2 / GLM-5）**：对于前端页面编写、常规接口开发、生成单元测试等，使用单价极低的国产第一梯队模型。这不仅能将成本压缩到极致，还能彻底规避合规与网络风险。
- **20% 高优核心任务（兜底/高智商突破：Claude Opus / GPT）**：针对极其复杂的 C++ 底层引擎重构、难以复现的内存泄漏等场景，网关有条件地允许部分核心员工路由至海外顶级模型，以最高智商攻坚难题。

### 3. 成本预算测算（以 40 人规模团队为例，Token 计费模型）
在按量后付费模式下，10 万元的年度预算非常充裕：
- **日均消耗预估**：假设每位研发每天极度重度使用，消耗 100 万 Token（输入+输出），团队每日总计消耗约 4000 万 Token。
- **费用核算**：
  - 若主力使用 DeepSeek/GLM 等（按 1~2元 / 百万Token计），每天仅需约 80 元，**全年约 2-3 万元**。
  - 混合 20% 的 Claude/GPT 高阶调用，每天增加约 200 元成本，**全年总预算可轻松控制在 5-10 万元以内**。
- **对比优势**：相比为每人购买 GitHub Copilot 个人版（每人每月约 10-20 美金，40 人全年约 7-14 万元），按量付费能大幅节省开支，且企业能完全掌控数据和模型选择权。

## 三、 内部 API 二次分发服务调研（管控与数据采集）

为了有效管理公司资产、防止额度滥用并留存审计数据，必须在内部私有云部署一个 **LLM API 网关服务**。经过调研，推荐以下成熟方案：

### 推荐方案：One API / New API（国内最主流的开源 API 分发管理工具）

- **统一协议，零感知切换**：将所有不同厂商的大模型 API 全部转换成标准格式。未来无论引入什么新模型，研发人员本地配置都不需更改。
- **精细化人员管控 (HR及管理层视角)**：
  - 可按前端/后端/移动端部门，或具体到每位员工，分配专属的内部 Token。
  - **绩效与习惯分析**：精确记录谁、在什么时间、高频使用了哪些 AI 能力，辅助 HR 和 Leader 了解团队工具接纳度。
- **额度防刷机制**：内部可为每个人设定硬性虚拟额度上限（例如每人每月最高消费 50 元）。到达上限即熔断，彻底杜绝因代码死循环或恶意调用导致的账单失控。

## 四、 后续采购后，风险控制与管理方法

引入大模型在带来效能飞跃的同时，也必须守住数据安全与责任归属的底线（法务、IT合规重点关注）：

### 1. 数据隐私与机密管控（Security & Privacy）
- **网关层脱敏拦截**：在内部网关前可增加轻量级的正则过滤清洗，严格禁止传入真实用户的敏感信息（PII如手机号、邮箱、密码 hash）以及公司的生产环境数据库账密。
- **防代码大面积泄露**：针对海外模型，严禁研发人员使用工具将整个项目的核心源码目录打包发往云端。规定 AI 仅限于“局部函数/单文件”上下文范围内的辅助。
- **制度护航**：建议 HR 协同研发管理层发布《AI 辅助编程安全规范》，新员工入职即签署，明确数据红线。

### 2. 系统可用性与“防封号”容灾（Reliability）
- **自动降级容灾 (Fallback)**：由于海外大模型 API 偶有网络抽风或针对国内 IP 的封控，我们在内部网关需配置“降级策略”。一旦首选模型（如 GPT）超时失败，网关将自动且无缝地将请求路由至备用模型（如 DeepSeek/GLM-5），确保研发人员的编程心流不被打断。

### 3. 代码质量把控与责任归属（Quality & Accountability）
- **AI 仅为副驾，责任在人**：需在内部达成共识，大模型生成的代码（尤其是 C++ 等内存敏感语言）存在“幻觉”风险。工具不为代码引发的生产事故背锅，研发人员仍是代码质量的第一责任人。
- **质量门禁（CI/CD）**：所有包含 AI 生成的代码在合并入主干前，必须由人类工程师进行逻辑确认（Code Review），并强制通过现有的自动化单元测试体系。
