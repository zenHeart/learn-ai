# Claude.ai 平台功能指南

> claude.ai 是 Anthropic 的核心产品界面，支持网页、手机 App、桌面 App 三端同步。作为 Pro 会员，你能使用远超免费版的功能集。本文系统梳理所有 Pro 专属或增强功能。

## 访问入口

| 平台 | 地址/方式 |
|------|---------|
| 网页版 | [claude.ai](https://claude.ai) |
| iOS App | App Store 搜索 "Claude" |
| Android App | Google Play 搜索 "Claude" |
| 桌面 App (macOS/Win) | [下载地址](https://claude.ai/download) |

所有平台共享同一账号、对话历史和项目数据。

## Projects 专属工作区

### 是什么

Projects 是 Claude.ai 中的独立工作区，每个 Project 有自己的：
- **聊天历史**：与该项目相关的所有对话
- **知识库**：上传的文档、代码、参考资料
- **自定义指令**：定义 Claude 在这个项目中的角色和行为规范

### 核心优势

**持久上下文**：普通对话每次都从零开始，Project 让 Claude 在每次对话时都能读取你上传的背景资料，无需重复粘贴。

**RAG 知识库（Pro 专属）**：上传大量文档后，Pro 用户享有 10 倍容量扩展，Claude 会自动检索相关片段来回答问题，而不是死记硬背全部内容。

**自定义角色**：在 Project 指令中写明 Claude 的角色，例如："你是一位熟悉我们公司技术栈的高级前端工程师，代码风格遵循我上传的规范文档。"

### 如何使用

1. 在左侧边栏点击 **+ 新建项目**
2. 给项目命名（如"前端重构项目"）
3. 点击 **添加到项目知识库** 上传相关文档
4. 在 **项目指令** 中写下 Claude 的角色定义和行为规范
5. 在项目内开始对话，Claude 始终带着这些上下文

### 支持的知识库文件类型

PDF、Word、Excel、TXT、Markdown、代码文件、CSV 等常见格式。单文件最大 10MB，项目总容量视计划而定（Pro 用户有大幅扩展）。

### 最佳实践

- **一个项目对应一个工作域**：如"产品设计"、"技术文档"、"市场研究"分别建立独立项目
- **上传规范文档**：代码规范、品牌手册、API 文档放入知识库，Claude 会自动参考
- **写好指令**：指令越具体，Claude 越能给出符合你期望的回答

## Extended Thinking 深度推理

### 是什么

Extended Thinking（扩展思考）是 Claude 4 和 Claude 3.7 Sonnet 的专属能力。启用后，Claude 在回答前会花更多时间**分解问题、制定方案、探索不同路径**，类似人类的"慢思考"。

你会看到一个 **"Thinking..."** 指示器和计时器，完成后可以展开查看 Claude 的推理过程摘要。

### 适合什么场景

✅ **适合**：
- 复杂数学推导和物理问题
- 竞赛级编程题
- 多步骤项目规划
- 深度文档分析和对比
- 需要权衡多方案的技术决策

❌ **不适合**（会浪费时间）：
- 简单问答（今天天气如何）
- 基础写作（帮我写个欢迎邮件）
- 常规代码补全

### 如何启用

1. 在模型选择器中选择 **Claude 4**（或 Claude 3.7 Sonnet）
2. 点击对话框左下角的 **搜索与工具**
3. 打开 **Extended Thinking** 开关

**注意**：启用 Extended Thinking 会开启一个新对话，无法在已有对话中途切换。

### 使用技巧

- 问题要具体清晰，模糊问题会让推理方向跑偏
- 展开"Thinking"区域了解 Claude 的推理逻辑，有助于你评估答案可靠性
- 对于耗时任务，这个模式会明显消耗更多使用额度

## Research 深度研究

### 是什么

Research 是 Claude 的**代理式联网研究**功能。与普通网络搜索不同，它能进行多轮相互关联的搜索，自主决定下一步要调查什么，最终给出带有可验证引用的完整答案。

类比：普通搜索是"帮你搜一次"，Research 是"帮你研究一个课题"。

### 适合什么场景

- 调研某个技术方案的优劣和行业现状
- 了解竞品功能和市场格局
- 收集某个话题的多方观点
- 查找最新的行业数据和报告
- 任何需要综合多个信息源的问题

### 如何使用

1. 在聊天界面左下角找到 **Research** 按钮（白色=未启用，蓝色=已启用）
2. 点击启用（**必须同时开启网络搜索**）
3. 提问，Claude 会自动展开多轮研究

若 Claude 没有主动使用 Research，可以明确要求："请使用 Research 工具深入调查这个问题。"

### 注意事项

- Research 功能会话消耗比普通对话更快（因为需要检索多个来源）
- 结果会附带来源引用，方便你验证
- 适用于 Pro、Max、Team、Enterprise 计划，支持网页版、Desktop 和手机 App

## 网络搜索

### 是什么

网络搜索让 Claude 能够获取**实时互联网信息**，突破训练数据截止日期的限制。

### 如何启用

在对话框左下角点击 **搜索与工具 → 网络搜索** 开启。

### 适用场景

- 最新新闻和事件
- 实时数据（股价、天气、汇率）
- 近期发布的技术文档
- 任何可能在训练截止日期之后发生变化的信息

### 与 Research 的区别

| | 网络搜索 | Research |
|--|:--:|:--:|
| 搜索轮次 | 1-2 次 | 多轮自主探索 |
| 适合场景 | 快速查询事实 | 深度调研课题 |
| 时间消耗 | 几秒 | 几分钟 |
| 额度消耗 | 较少 | 较多 |

## 交互式图表与可视化

> 2026 年 3 月新功能：Claude 现在可以生成真正可交互的图表、图形和数据可视化，而不仅仅是静态图片。

### 是什么

Claude 可以直接在对话中生成**可交互的可视化内容**，用户可以在界面内直接操作：缩放、筛选、悬停查看详情、动态更新数据等。

### 支持类型

- **交互式折线/柱/饼图**：数据点可悬停查看，支持缩放和平移
- **流程图 / 思维导图**：节点可拖拽，支持折叠展开
- **数据仪表盘**：多图联动，实时过滤
- **网络关系图**：节点连接可视化，支持力导向布局

### 使用方法

直接描述需求，Claude 会判断是否生成交互式内容：

```
帮我把这份销售数据做成可以按月份筛选的交互式柱状图
```

```
画一个可以折叠的组件树结构图，展示 React 应用的组件层级
```

生成后在 Artifact 窗口直接交互，也可以导出为独立 HTML 文件。

## Artifacts 独立内容输出

### 是什么

Artifacts 是 Claude 在对话旁边**独立窗口中生成的内容**，与聊天分离展示，方便编辑、预览和导出。当内容通常超过 15 行，且能独立存在时会自动触发。

### 支持的 Artifact 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 文档 | Markdown 或纯文本 | 报告、文章、README |
| 代码 | 带语法高亮的代码片段 | React 组件、Python 脚本 |
| 网站 | 可预览的 HTML+CSS+JS | 落地页、表单、工具 |
| SVG 图像 | 矢量图形 | 图标、流程图 |
| 图表/流程图 | Mermaid 语法渲染 | 时序图、流程图、ER 图 |
| React 组件 | 可交互的前端组件 | 数据可视化、计算器 |

### 如何使用

- **编辑迭代**：直接要求 Claude 修改，变更实时显示在 Artifact 窗口
- **版本切换**：右上角的版本选择器可以回溯历史版本
- **导出**：右下角可查看源码、复制到剪贴板、下载文件
- **多个 Artifact**：一次对话可生成多个，通过滑块图标切换

### AI 驱动的 Artifacts（高级功能）

Pro 用户可创建**嵌入了 Claude 智能的应用**：
- Artifact 内部可调用 Claude API，实现真正的交互逻辑
- 通过 MCP 集成连接 Asana、Google Calendar 等外部服务
- 支持持久化数据存储（每个 Artifact 最大 20MB）
- 分享给他人使用时，对方使用自己的账户，不消耗你的额度

## Memory 跨对话记忆

### 是什么

Memory 让 Claude 能跨对话记住你的偏好、背景信息和工作习惯，不再每次都要重新介绍自己。

### 两种记忆类型

**用户主动记忆**：你明确告诉 Claude 记住某事：
```
记住：我是一个前端工程师，主要用 React + TypeScript，
偏好函数式编程，不喜欢 class 组件。
```

**自动记忆**（Claude Code 中）：Claude 在工作时自动学习并保存，如构建命令、调试偏好、常用工具配置。

### 如何管理记忆

- 进入 **账户设置 → 记忆** 查看所有已保存的记忆
- 可以编辑或删除不需要的条目
- 支持导入/导出记忆（便于备份或在多设备同步）

## Skills 技能系统

### 是什么

Skills 是"包含指令、脚本和资源的文件夹，Claude 动态加载来提升专项任务性能"。简单说：技能让 Claude 在特定任务上更专业、更稳定。

### 四种技能类型

**Anthropic 内置技能**：处理 Excel、Word、PowerPoint、PDF 等文档任务时自动调用，无需手动操作。

**自定义技能**：你或团队创建的专属工作流，如品牌风格指南应用、标准化邮件模板、会议记录整理格式。

**合作伙伴技能**：来自 Notion、Figma、Atlassian 等产品的预构建技能，配合对应连接器使用。

**组织技能（Team/Enterprise）**：管理员为全团队统一部署的标准化工作流。

### 如何使用

在账户 **自定义** 菜单中的 **技能** 目录，可以查看、启用和管理所有可用技能。使用时，Claude 会自动识别当前任务是否有对应技能并加载，也可以手动用 `/技能名` 调用。

### 如何创建自定义技能

详见官方文档：[如何创建自定义技能](https://support.claude.com/zh-CN/articles/12512198)。核心是：用 Markdown 写指令文件，可选配 Python 脚本，放入规定目录结构。

## 个性化设置

### 自定义样式（Response Style）

在 **设置 → 自定义样式** 中定义 Claude 的回复风格：

- **简洁模式**：只给答案，不要解释
- **教学模式**：详细说明每个步骤的原因
- **技术模式**：使用专业术语，面向开发者
- **创意模式**：更有创意和个性的表达

可以用自然语言描述你期望的风格，Claude 会应用到后续对话。

### 语言设置

Claude 默认用你提问的语言回答。如果你希望 Claude 始终用特定语言（如简体中文）回答，即使你用英文提问，可以在系统提示或项目指令中指定。

### 外观设置

在 **设置 → 外观** 中切换：
- 亮色/暗色/跟随系统
- 字体大小

### 模型切换

点击对话框右上角或设置中的模型选择器，可以在以下模型间切换：

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| Claude Opus 4.6 | 最强推理、最高质量，世界最强编程模型 | 复杂任务、深度分析、企业级代理 |
| Claude Sonnet 4.6 | 前沿智能与速度平衡，企业工作流首选 | 日常开发、写作（默认） |
| Claude Haiku 4.5 | 最快响应，接近前沿智能 | 简单问答、批量处理 |

## 隐私与安全

### 隐私聊天

**隐私聊天**是一种特殊对话模式，在此模式下的内容：
- 不会被用于 Anthropic 的模型训练
- 不会出现在你的对话历史中（本地和云端都不保存）
- 会话结束后彻底清除

启用方式：在新建对话时选择 **隐私聊天** 选项。

**适用场景**：讨论敏感商业信息、个人隐私内容、竞争情报等不希望留存记录的对话。

### 数据导出

在 **账户设置 → 数据控制** 中可以导出你的所有对话历史和账户数据，格式为 JSON。

## 对话管理

### 分享对话

已完成的对话可以生成分享链接，发给他人查看（只读，对方无法继续对话）。在对话右上角点击 **分享** 图标，可以设置是否公开。

### 搜索历史对话

左侧边栏顶部有搜索框，可以全文搜索历史对话内容，快速找到之前讨论过的话题。

### 上传文件

在对话框中可以直接拖拽或点击附件图标上传文件。支持文档、代码、图片、PDF 等格式，Claude 会分析文件内容并基于其回答问题。

## 使用量管理

### 理解使用限制

Pro 用户享有 **Standard** 级使用量（远高于 Free 的 Limited），但仍有上限（Max 5x 计划是 Pro 的 5 倍）。使用量按以下顺序消耗更快：
1. 普通对话（最省）
2. 开启网络搜索的对话
3. Extended Thinking 对话
4. Research 对话（最耗）

### 如何查看使用情况

在 **设置 → 使用情况** 中可以查看当前周期的额度消耗情况。

### 使用量套餐

如果你的 Pro 额度不够用，可以购买额外的使用量套餐（按量付费），无需升级到 Max 计划。进入账户设置查看套餐选项。

### 最佳实践：省额度的用法

- 只在必要时启用 Extended Thinking 和 Research
- 将重复性问题做成项目知识库，避免每次粘贴相同背景
- 简单问题直接问，不要加过多上下文
- 用 Claude Haiku 处理简单批量任务

## 实用场景示例

### 场景 1：技术调研（Research + Projects）
```
1. 创建 Project："技术选型调研"
2. 上传现有系统架构文档
3. 启用 Research 功能
4. 问：请调研 React Server Components vs 传统 CSR 在我们这类电商场景下的适用性，
   结合我上传的架构文档给出建议。
```

### 场景 2：复杂问题规划（Extended Thinking）
```
1. 选择 Claude 4，启用 Extended Thinking
2. 问：我需要将一个 jQuery 项目迁移到 React，
   项目有 200 个页面，团队 5 人，需要 3 个月完成。
   请帮我制定详细的迁移策略和风险控制方案。
```

### 场景 3：生成可交互原型（Artifacts）
```
问：用 React 帮我做一个数据表格组件，支持排序、筛选、分页，
使用 Tailwind CSS 样式，数据从 props 传入。
```
→ Claude 生成 Artifact，右侧直接预览效果，不满意可继续迭代。

### 场景 4：个性化写作助手（Projects + 自定义指令）
```
项目指令：
你是一位技术写作专家，帮我写面向前端开发者的技术博客。
文章风格：直接、有深度、带实际代码示例。
避免：过多废话、过于学术化的表达。
读者假设：有 2 年以上 React 经验的开发者。

知识库：上传你过去的文章作为风格参考
```
