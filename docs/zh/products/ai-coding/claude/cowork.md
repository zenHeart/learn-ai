# Cowork 完整指南

> Cowork 是 Claude Desktop 内置的**桌面代理模式**，让 Claude 直接读写你的本地文件、操控应用程序、执行自动化任务。它是 claude.ai 对话模式的进化——从"帮你想"升级为"帮你做"。

## 什么是 Cowork

Cowork 不是聊天，而是一个**任务执行代理**。核心区别：

| | Claude.ai 对话模式 | Cowork 代理模式 |
|--|:--:|:--:|
| 文件访问 | 需手动上传 | 直接读写本地文件 |
| 任务长度 | 单轮对话 | 长时间复杂任务 |
| 自动化 | 不支持 | 支持定期调度 |
| 输出 | 文字/代码 | Excel、PPT、文档、邮件等 |
| 多任务 | 串行对话 | 并行分解工作流 |

**适用人群**：Pro、Max、Team、Enterprise 付费用户，需安装 Claude Desktop（macOS / Windows）。

---

## 核心能力

### 本地文件直接访问
Claude 可以读取和写入你授权的文件夹，无需手动上传或下载。典型场景：
- 整理 Downloads 文件夹中的文档
- 读取本地 CSV 并生成分析报告
- 批量重命名文件
- 读取代码目录并生成技术文档

### 专业文档生成
Cowork 内置文档技能，可直接生成并保存到本地：
- **Excel 表格**：数据分析、财务模型、甘特图
- **PowerPoint 演示文稿**：自动布局、图表嵌入
- **Word 文档**：报告、合同、说明书
- **PDF**：合并、拆分、填写表单

### 任务分解与并行
Claude 将复杂工作拆分为子任务并协调执行，例如"分析上个季度所有会议记录，提取关键决策，生成摘要报告"会被分解为：读取文件 → 逐个分析 → 汇总整合 → 写入输出文件。

### 计划任务调度
创建按频率自动运行的任务，如每日早报、每周汇总、定期文件整理。

## 快速上手

### 第一步：安装并打开 Claude Desktop

下载地址：
- [macOS](https://claude.ai/api/desktop/darwin/universal/dmg/latest/redirect)
- [Windows x64](https://claude.ai/api/desktop/win32/x64/exe/latest/redirect)

安装后登录你的 Claude Pro 账户。

### 第二步：切换到 Cowork 模式

打开 Claude Desktop 后，在顶部标签栏切换到 **Cowork** 选项卡（与普通聊天标签并列）。

### 第三步：授权文件夹

第一次执行文件相关任务时，Cowork 会请求访问特定文件夹的权限。建议：
- 只授权需要操作的目录（不要授权整个磁盘）
- 避免授权含有密码、密钥、财务凭证的目录

### 第四步：描述你的任务

直接用自然语言描述需要完成的工作：

```
帮我读取 ~/Downloads/会议记录/ 文件夹中所有 .docx 文件，
提取每个文件中的"行动项"，汇总到一个 Excel 表格里，
保存为 ~/Documents/行动项汇总.xlsx
```

Claude 会展示执行计划，确认后开始运行。

## 定期任务自动化

### ⚠️ 两种任务类型，关键区别

| | 桌面定期任务 | 云端定期任务 |
|--|:--:|:--:|
| **运行环境** | 你的电脑 | Anthropic 云服务器 |
| **电脑关机能运行？** | ❌ 必须开机 | ✅ 随时运行 |
| **访问本地文件？** | ✅ 直接访问 | ❌ 仅限云端 |
| **创建入口** | Cowork 侧边栏 | claude.ai/code 或 /schedule |
| **适合场景** | 处理本地文件的任务 | 不依赖本地文件的任务 |

**桌面任务**：电脑必须唤醒且 Claude Desktop 保持打开。若休眠，任务跳过（不补跑）。
**云端任务**：运行在 Anthropic 管理的服务器上，24 小时可用，适合不需要本地文件的自动化。

### 什么是定期任务

定期任务是保存下来的提示词 + 执行计划，可以按频率自动运行，或按需手动触发。

### 常见使用场景

- **每日早报**：汇总过去 24 小时的邮件、日历事件，生成今日待办
- **周报整理**：每周五将本周工作日志整理成结构化报告
- **定期研究**：每天追踪特定行业新闻，生成摘要文件
- **文件整理**：每周对指定文件夹排序清理
- **数据更新**：定期拉取数据，更新 Excel 报表

### 创建方法一：用 /schedule 命令

在任意 Cowork 任务对话中输入 `/schedule`，Claude 会引导你完成创建：

```
/schedule
```

Claude 会提问：任务名称、执行频率、是否绑定特定文件夹等。

### 创建方法二：从定期任务页面

1. 点击左侧边栏的 **定期任务**
2. 点击右上角 **+ 新建任务**
3. 填写表单：

| 字段 | 说明 |
|------|------|
| 任务名称 | 便于识别的名称，如"每日工作早报" |
| 描述 | 简短说明任务用途 |
| 执行提示 | Claude 每次运行时收到的完整指令 |
| 运行频率 | 每小时 / 每天 / 每周 / 工作日 / 手动 |
| 模型选择 | 默认 Claude Sonnet，复杂任务可选 Opus |
| 工作文件夹 | 可选，绑定特定目录 |

### 管理定期任务

- **手动触发**：点击任务旁边的运行按钮，立即执行一次
- **暂停/恢复**：切换任务的启用状态
- **查看历史**：点击任务查看历次运行记录和输出

## 插件系统

### 什么是插件

插件是将**技能（Skills）+ 连接器（Connectors）+ 子代理（Agents）**打包在一起的扩展包，安装一次即可获得一整套专项能力，无需逐个配置。

例如安装"财务分析"插件后，Claude 就具备了：
- 读取财务报表的技能
- 连接特定财务 API 的连接器
- 专门处理财务分析的子代理

### 安装插件

1. 打开 Claude Desktop，切换至 Cowork 选项卡
2. 点击左侧边栏的 **自定义** 菜单
3. 选择 **浏览插件** 查看可用选项
4. 点击 **安装** 完成部署

也可以上传本地自定义插件文件（`.skill` 格式）。

### 使用插件技能

安装插件后，在 Cowork 任务对话中：
- 输入 `/` 查看所有可用技能列表
- 点击 `+` 按钮浏览并调用技能
- 直接描述任务，Claude 会自动匹配合适的技能

### 自定义插件

点击已安装插件旁边的 **自定义** 按钮，可以与 Claude 协作调整插件的提示词、技能和连接器配置，使其更适合你的工作流程。

### 构建自己的插件

Cowork 内置 **Plugin Creator** 插件，可以指导你从零开始构建：

```
/plugin-create
```

按照引导完成：插件名称 → 技能定义 → 连接器配置 → 测试验证 → 导出分享。

## Claude Desktop 代码标签页（Claude Code）

Claude Desktop 除了 Cowork 之外，还有一个**代码（Code）标签页**，是面向开发者的 Claude Code 图形界面，提供比终端 CLI 更多的视觉能力：

| 功能 | 说明 |
|------|------|
| **可视化 Diff 审查** | 直观查看代码变更，支持内联注释 |
| **App 实时预览** | 启动开发服务器，在侧边实时预览效果 |
| **PR 监控** | 实时查看 CI 状态，开启 Auto-fix 让 Claude 自动修复 |
| **并行会话** | 多个 Claude 实例同时处理不同任务，Git worktree 自动隔离 |
| **Dispatch** | 从手机发送任务，桌面自动创建会话执行 |
| **Connectors** | 连接 GitHub、Slack、Linear 等外部服务 |
| **SSH 会话** | 连接远程服务器上的开发环境 |
| **云会话** | 启动运行在 Anthropic 云上的长期任务（电脑关机也能跑） |

> Cowork（任务代理）和 Code（编程代理）是 Claude Desktop 中的两个并列标签页，面向不同使用场景。

## 计算机使用（Computer Use）

### 什么是计算机使用

Computer Use 让 Claude 能够**像人一样操作你的电脑**——控制鼠标、键盘、查看屏幕，从而操作任何桌面应用和浏览器，即使没有 API 接口。

典型场景：
- 在浏览器中搜索、填写表单、提取信息
- 操作 Excel、Word 等桌面软件
- 在多个应用之间传递数据（如从邮件提取数据填入 CRM）

### 如何启用

在 Claude Desktop 设置中，进入 **Desktop App → Computer Use** 开启。

**注意**：启用后 Claude 在访问每个应用前会请求单独权限，你可以选择性授权。

### 安全边界

Computer Use 权限较高，建议：
- 执行前仔细查看 Claude 的操作计划
- 不要在涉及网银、密码管理器等敏感应用中使用
- 监控任务过程，随时可以中断

## 跨应用协作

### 从任意地方派发任务

Cowork 支持不打开 Claude Desktop 也能发起任务：

**方法**：通过 Claude 的快捷方式或全局菜单，从任意应用发送任务到 Cowork 队列，Claude Desktop 会在后台接收并执行。

适用场景：在浏览器看到一篇文章想要总结并保存 → 选中文本 → 发送给 Cowork → Claude 自动处理并保存到指定文件夹。

### 跨应用工作流示例

```
场景：将本周所有电子邮件中的会议约定汇总到日历文件

Claude 执行步骤：
1. 访问邮件客户端（Computer Use）读取本周邮件
2. 识别包含时间、地点、参与者的会议信息
3. 整理成结构化数据
4. 写入本地日历文件或生成 .ics 文件
```

## 安全使用指南

Cowork 拥有访问你本地文件和应用的权限，请了解以下安全原则。

### Anthropic 的保护机制

- **注入攻击防护**：Claude 被训练识别并拒绝恶意内容中的欺骗指令
- **内容分类**：自动扫描不受信任的内容，标记潜在风险
- **删除保护**：永久删除文件前必须获得你的明确许可
- **按应用授权**：Computer Use 访问每个应用前单独请求权限

### 你需要做的

**选择性授权**：只给 Claude 访问必要文件夹的权限，避免授权含有财务信息、登录凭证的目录。

**监控任务**：关注意外行为——Claude 访问了你没提到的文件或网站吗？

**谨慎设置定期任务**：
- 从简单任务开始，逐步扩展
- 不要自动化涉及敏感数据、不可逆操作（如发送邮件、删除文件）的任务
- 先手动跑几次确认结果正确，再开启自动调度

**限制浏览器访问**：网络内容是提示词注入的主要途径，将 Computer Use 的浏览器访问限制在受信任的网站。

**审查插件权限**：安装插件前仔细阅读其权限请求，只安装来源可信的插件。

### 你的责任边界

> Claude 代表你执行的所有操作，由你承担最终责任，包括发布的内容、产生的交易、修改的数据。

## 常见问题

**Q：Cowork 任务执行时我需要一直盯着吗？**

不需要。你可以设置任务后离开，Claude 会在后台执行。完成后会通知你查看结果。但对于高风险操作（如批量删除、发送邮件），建议手动确认。

**Q：定期任务如果我的电脑关机了怎么办？**

任务会被跳过，下次触发时间到来时正常运行。不会"补跑"错过的次数（例如你出差 3 天，每日任务不会一次性跑 3 遍）。

**Q：Cowork 和 Claude Code 有什么区别？**

| | Cowork | Claude Code |
|--|:--:|:--:|
| 主要用户 | 非开发者/通用用户 | 开发者 |
| 操作对象 | 文件、应用、浏览器 | 代码库、终端、Git |
| 使用界面 | Claude Desktop 图形界面 | 终端/IDE |
| 自动化方式 | 定期任务 | 无头模式/CI |

**Q：插件连接器能访问防火墙后面的服务吗？**

标准连接器通过 Anthropic 云服务连接，自定义连接器需要指向可公开访问的服务器。内网服务需要参考网络要求文档单独配置。
