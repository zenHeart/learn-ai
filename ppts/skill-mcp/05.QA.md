# 05. QA 问答环节

---
layout: default
---

### **Q1: 为什么非要套一层 MCP 增加复杂度？我直接在脚本里调 API 不是更简单吗？**

A: **对于"一次性脚本"，直接调 API 确实更简单；但对于"生态级集成"，MCP 是降维打击。**

<div class="mt-6 space-y-5 text-sm">

- **复利效应**: [Anthropic 在官方博客指出](https://www.anthropic.com/news/model-context-protocol)，只要你写了一次 Jira 的 MCP Server，明天不管是 Cursor、Windsurf、Claude 甚至后天的某个新 IDE 出现，**不需要改一行代码**，所有的 AI 工具都能瞬间具备操作 Jira 的能力。
- **权限收口**: 如果直接让 AI 写 Python 脚本去调数据库，意味着你要把生产环境的账号密码直接暴露给大模型（极度危险）。而 MCP 架构把密钥和具体执行逻辑封装在本地 Server 里，对大模型完全隔离。

</div>

---

### **Q2: 为什么我写的 Skill，AI 经常乱用或者根本不听我的规则？**

A: **因为你把它当成了人，而不是当成了编译器。**

<div class="mt-6 space-y-5 text-sm">

- **官方铁律**: 根据 [Anthropic 的 Tool Use 最佳实践](https://docs.anthropic.com/en/docs/build-with-claude/tool-use)，不要对 AI 说"请尽量避免使用..."，而是要说"**绝对禁止**在 XX 情况下使用，否则会导致严重错误"。
- **Schema 约束**: 把所有能用 `enum` (枚举) 框死的地方全部框死。如果你让 AI 传一个自由的字符串，它就会产生幻觉。给它明确的下拉菜单，它就不会出错。

</div>

---
layout: two-cols
---

### **Q3: 自己开发 MCP Server 门槛高吗？**

A: **非常低，几乎等同于写一个普通的 Node.js / Python 函数。**

<div class="mt-6 space-y-4 text-sm">

1. **SDK 完善**: 官方提供了 `@modelcontextprotocol/sdk` (TS) 和 `mcp` (Python)
2. **特别是 Python 的 FastMCP**: 它的心智模型完全借鉴了 FastAPI。你只需要写一个普通的 Python 函数，加一个 `@mcp.tool()` 装饰器，它就自动变成了一个标准的 MCP Tool

</div>

::right::

### **Q4: `.cursorrules` (规则) 和 CLI 里的 Skill 到底有什么区别？**

A: **触发机制和作用范围不同。**

<div class="mt-6 space-y-4 text-sm">

1. **`.cursorrules` (被动守则)**: 只要在这个项目里写代码，它就会作为 Background Context 一直存在，默默约束 AI 的所有行为
2. **Skill (主动大招)**: 是一套包含具体步骤的"工作流"。它平时潜水，只有当你敲下 `/audit` 或者 `/deploy` 时才被唤醒，去执行一套复杂的 SOP

</div>

---

### **Q5: MCP 只能用于本地（stdio）吗？可以跨网络调用吗？**

A: **完全可以跨网络。**

<div class="mt-6 space-y-4 text-sm">

1. **企业内网部署**: 你可以把 MCP Server 部署在公司内网的一台服务器上（例如 10.0.0.1），然后使用 **SSE (Server-Sent Events)** 作为 Transport 层暴露
2. **集中管控**: 全体研发的 Cursor/Claude 都可以通过 HTTP 连接到这个统一的节点。这不仅方便版本控制，还能在服务端统一做权限审计

</div>

::right::

### **Q6: 遇到复杂的流水线，我是写一个大 Skill，还是写多个小 Skill？**

A: **坚决写多个小 Skill。**

<div class="mt-6 space-y-4 text-sm">

1. **控制变量**: 像写微服务一样写 Skill。一个叫 `run-linter`，一个叫 `build-image`
2. **编排机制**: 让一个高等级的 Agent 去根据当前状态，按需调用这些原子化的 Skill。把它们揉成一团只会导致 AI 经常在中间某一步因为超时而全盘崩溃

</div>

---

### **Q7: MCP + Skill 组合使用的最佳实践是什么？**

A: **MCP 提供"武器"，Skill 定义"打法"。**

<div class="mt-6 space-y-4 text-sm">

1. **MCP 负责对外**: 封装对数据库、API、文件系统等外部资源的访问能力
2. **Skill 负责对内**: 定义团队规范、SOP 流程、错误处理策略
3. **Agent 负责调度**: 由 AI 根据任务复杂度，自动选择调用哪些 MCP 和 Skill

</div>
