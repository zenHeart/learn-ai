# 02.2 Skill 核心

> **本质上，Skill 就是大模型的 Tool Use (Function Calling) 的封装，再加上特定的上下文规则约束。**

## 演进路线：从硬编码到 AI 编排

<div class="grid grid-cols-2 gap-8 mt-6">

<div class="p-4 border border-red-200 rounded-lg bg-red-50/30">

### ❌ 过去 (硬编码工作流)
- 比如 ESLint 或 CI/CD 脚本
- 只有 If-Else，遇到异常直接抛错退出
- "人工定义每一步怎么走"

</div>

<div class="p-4 border border-green-200 rounded-lg bg-green-50/30">

### ✅ 现在 (Skill / Tool Use)
- 给 AI 提供"锤子"（Tool）和"说明书"（Description/Schema）
- AI **自适应地决定**什么时候用锤子，怎么用
- 从"死板脚本"进化为"懂变通的 AI 流水线"

</div>

</div>

# 构建优质 Skill 的三大要素

<div class="grid grid-cols-3 gap-4 mt-6">

<div class="bg-blue-50 dark:bg-blue-900/30 p-4 rounded-lg border border-blue-200">

### 1. 清晰的边界 (Description)
告诉 AI 什么时候该用
- ❌ "用于分析代码"
- ✅ "当用户要求分析安全漏洞时调用，**仅支持 `.ts` 和 `.js` 文件**"

</div>

<div class="bg-purple-50 dark:bg-purple-900/30 p-4 rounded-lg border border-purple-200">

### 2. 严格的入参 (Schema)
约束大模型的发挥空间
- 使用 Enum 限制选项
- Required 字段避免缺失
- 生产环境开启 `strict: true`

</div>

<div class="bg-green-50 dark:bg-green-900/30 p-4 rounded-lg border border-green-200">

### 3. 兜底与容错 (Fallback)
失败了怎么办？
- API 超时不要陷入死循环
- "报错超过 2 次立即停止并报告"

</div>

</div>

# Schema 设计：正反对比

## ❌ 错误示例

```yaml
input_schema:
  properties:
    query:
      type: "string"  # 太模糊
```

## ✅ 正确示例

```yaml
input_schema:
  properties:
    category:
      type: "string"
      enum: ["tech", "life"]  # 限制可选值
  required: ["query"]
  strict: true
```
