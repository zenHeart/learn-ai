# 04. 实战演练

> 理论很丰满，落地到底能干啥？看看这些直击研发痛点的场景。

---

## 场景 1：数据库直接生成 CRUD 代码 (MCP)

**痛点**: 新建一个表后，要手动写 Entity、DTO、Controller、Service，繁琐且容易出错。

**解法**: 用 `@modelcontextprotocol/server-postgres` 挂载本地数据库。

**演示工作流**:
1. 在 Cursor/Claude 中配置 Postgres MCP Server
2. 输入提示词："请查看 `users` 和 `orders` 表的结构（schema），并为我生成完整的 NestJS CRUD 接口代码，包含 Swagger 装饰器。"
3. **效果**: AI 自动调用 `query_database` 工具查询表结构，然后一次性生成 100% 准确的 TS 代码。

---

## 场景 2：打通 Jira/GitLab 自动提 PR (MCP)

**痛点**: 修复 Bug 需要切分支、查 Jira 单号，写 Commit Message，最后还要去浏览器里提 PR。

**解法**: 开发一个对接内部 Jira 和 GitLab 的企业级 MCP Server。

**演示工作流**:
1. 开发者在终端输入：`修复了登录页的空指针报错，请帮我提交代码并关联 Jira 单号 WEB-1024`
2. **AI 自动执行**:
   - 调用 `jira_get_issue` 获取 WEB-1024 的详情
   - 调用本地 `bash` 工具创建分支、Git commit
   - 调用 `gitlab_create_mr` 创建合并请求
3. **效果**: 整个流水线被 AI 一键接管

---

## 场景 3：把前端代码规范转化为 `.cursorrules` (Skill)

**痛点**: 团队规定了严格的 React Hooks 使用规范和文件命名，代码审查经常因为格式打回。

**解法**: 在项目根目录下發 `.cursorrules`

```markdown
# Front-end Project `.cursorrules`

## 1. 核心技术栈
- React 18 (严格遵循 Functional Component 范式)
- TypeScript (禁止使用 `any`，必须定义 Interface)

## 2. 状态管理规范 (强制)
当用户要求新建一个跨组件共享状态时：
- ❌ 绝对不要引入 Redux 或 Context API
- ✅ 必须使用 Zustand
```

<strong>降维打击</strong>: 以后只要研发按下 `Cmd+K` 让 AI 写代码，AI 自动受此规则约束
</div>

## 场景 4：结合 MCP 编排"发版验收"工作流 (MCP + Skill)

**痛点**: 每次发版前都要手动跑单测、查包大小、最后还要切去终端打 Git Tag。

**解法**: 编写一个复合能力的 Claude Skill

```yaml
# ~/.claude/skills/release-check.md
---
name: release-check
description: 执行上线前的最终全量检查
allowed-tools: [bash, git, mcp_jira]
---

当用户输入 `/release` 时，请严格按照以下顺序执行：
1. 运行 `npm run test:coverage`。如果覆盖率低于 80%，立即报错并停止
2. 运行 `npm run build`，检查 `dist` 目录大小是否超过 5MB
3. 调用 `mcp_jira` 工具，读取当前绑定的 Jira Issue 状态
4. 一切通过后，使用 bash 运行 `git tag v{版本号}`
```

<div class="mt-4 p-4 bg-purple-50 text-purple-800 rounded-lg text-sm border border-purple-200">
<strong>价值升维</strong>: 我们不再是教 AI "写代码"，而是在教 AI "当研发主管"
</div>

## 开发自定义 MCP Server 实战 (Node.js)

如果官方没有，如何自己写一个？

```typescript
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

// 1. 初始化 Server
const server = new McpServer({ name: "my-internal-tools", version: "1.0.0" });

// 2. 注册 Tool (AI 的"手")
server.tool(
  "query_jira",
  { issueId: z.string().describe("Jira 任务编号，如 PROJ-123") },
  async ({ issueId }) => {
    const data = await fetchJiraTicket(issueId);
    return { content: [{ type: "text", text: JSON.stringify(data) }] };
  }
);

// 3. 启动通信层 (stdio)
const transport = new StdioServerTransport();
await server.connect(transport);
console.log("MCP Server running on stdio");
```
