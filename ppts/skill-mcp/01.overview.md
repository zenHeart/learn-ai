<div class="punchline mt-6">

# 三个真实的研发痛点

<div class="mt-6 grid grid-cols-3 gap-4">

<div v-click="1" class="chapter-card">

### 🔌 连不进系统

* 写完代码不会自己跑 CI
* 改 bug 不知道 Jira 单号
* 看不到内网数据库 schema

</div>

<div v-click="2" class="chapter-card">

### 📋 守不住规范

* 团队约定的 commit 格式没人遵守
* Code review 标准靠口头传
* 同一个错误反复犯

</div>

<div v-click="3" class="chapter-card">

### 🔁 接不上工序

* 测试 → 打 tag → 发 Slack 全靠手
* 跨工具切来切去
* 每次都从零提示

</div>

</div>

<div v-click="4" class="mt-8 p-4 theme-combo">

**两条腿才能走路：**
**MCP** 给 AI 装上「**手脚**」—— 让它能伸进内部系统；
**Skill** 给 AI 装上「**肌肉记忆**」—— 让它会做团队的活。

</div>

关键是要让听众"对号入座"：每个痛点都对应一个本期要交付的解决方案。

# 课程目标 · 你能带走什么

<div class="mt-4 grid grid-cols-2 gap-6">

<div v-click="1" class="theme-mcp">
