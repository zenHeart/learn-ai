# 1. 概述

> **Vibe Coding** 是一种利用 AI 编程工具构建软件的开发范式。

::left::


通俗来讲，你的核心职责从构建代码，转向更高层面的架构设计/评估/审查事项，而把构建代码的细节交给 AI 模型。

::right::

# AI 编程层级

| 等级 | 描述 |
| :--- | :--- |
| L1 | 还没有接触到 AI 编程 |
| L2 | 装了 AI 插件，多用来实现代码补全 |
| L3 | 开始信任 AI 编程，对话式完成任务 |
| L4 | 使用 YOLO 模式，只对 AI 最终结果进行审查 |
| L5 | 纯命令行交互，只关注 AI 的完成进度 |
| L6 | 开始多任务模式，每个窗口管理各自的任务 |
| L7 | 管理多达 10 个以上的窗口让他们各司其职 |
| L8 | 采用任务编排，并行管理多个任务，让 AI 自行协调多 Agent 的开发，你只交代任务、验收结果 |

</div>

# 本章小结：Vibe Coding 概述

<v-clicks>

1. **Vibe Coding 是编程范式迁移** 核心职责从**构建代码**，转向更高层面的**架构设计/评估/审查等工作**, 而将构建工作托付给 AI
2. **能力层级演进 L1-L8** 从被动的 **代码补全** 进化到主动的 **多 Agent 编排**
3. **工具生态认知 (L1-L5)** 从**代码补全** (如 Copilot) 到 **AI Agent 团队**(如 [Claude Code](https://code.claude.com/docs/en/agent-teams))

</v-clicks>
