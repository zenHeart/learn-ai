# [AGENTS.md](https://agents.md/)

> 给 AI 看的 README.md，为 AI 提供整个项目的核心上下文和规则，该内容会作为系统上下文每次对话都携带 <VibeExample id="3.1" />

::left::

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- 放置在项目根目录 [AGENTS.md](https://cursor.com/docs/context/rules#agentsmd) 


```markdown
# 架构与规范
- 统一使用 TypeScript 严格模式
- 必须补齐 React PropTypes
- 状态管理必须使用 Pinia
```

::right::

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- 使用专有的 [CLAUDE.md](https://code.claude.com/docs/en/best-practices#write-an-effective-claude-md)（通过 `/init` 指令交互式创建）, [github issue](https://github.com/anthropics/claude-code/issues/6235) 建议在 CLAUDE.md 中通过 `@AGENTS.md` 引用 AGENTS.md

```bash
# claude
/init # 初始化 CLAUDE.md
# 在创建的 CLAUDE.md 中通过 @AGENTS.md 引用 AGENTS.md
```

# 细粒度模块管控 (Rules)

> 项目级规则像宪法，目录级规则像地方法规。通过文件路径匹配，让 AI 在不同目录下自动应用不同的规范。 <VibeExample id="3.2" />

::left::

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- 在 `.cursor/rules/*.mdc` 目录管理 [Project Rules](https://cursor.com/docs/context/rules#project-rules)。
- 利用 YAML Frontmatter 设定匹配目标通配符（Globs），按需触发。

```yaml
---
description: React 组件开发守则
globs: src/components/**/*.tsx
alwaysApply: false
---
只能使用函数式组件 (Hooks)，禁止编写 Class。
```

::right::

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- 存放在 `.claude/rules/` 文件夹下的独立 [Modular Rules](https://code.claude.com/docs/en/memory#modular-rules-with-claude/rules/)。
- 通过 paths 指定匹配目标通配符（Globs），按需触发。

```markdown
---
paths:
  - tests/**/*.{ts,tsx}
  - tests/**/*.spec.ts
---

# 最佳实践
当你在 `tests/` 下编写用例时，请默认使用
Vitest 框架，而非 Jest。
```

# 工作流沉淀 (Commands & Skills)

> 将高频操作（如代码 review、 SOP 流程）封装为命令或者技能，实现复用 <VibeExample id="3.3" />

::left::

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- 在项目本地 `.cursor/commands/*.md` 目录创建 [Slash Commands](https://cursor.com/docs/context/commands)，通过 `/xxx` 斜杠快速呼出
- 通过 `.cursor/skills/` 目录定义 [Skills](https://cursor.com/docs/context/skills)，每个 skill 为一个文件夹、内含 `SKILL.md`，可与 .claude、Codex 等格式兼容，供 AI 调用

```markdown
---
description: 代码安全审查
globs: src/**/*
alwaysApply: false
---

请审查当前文件的代码问题，包括：
- 安全漏洞
- 性能问题
- 代码规范
```

::right::

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- 在项目本地 `.claude/skills/*.md` 创建 [Skills](https://code.claude.com/docs/en/skills)，支持通过 YAML Frontmatter 定义可用工具
- Skills 可被其他 Agent 调用，实现工作流复用

```yaml
---
description: 代码安全扫描
allowed-tools: [Read, Grep]
# 可选：model: sonnet
# 可选：max-turns: 10
---
请扫描依赖树，全量报告 CVE 漏洞。
```

# 外部能力接入 (MCP)

> **Model Context Protocol (MCP)**：标准化的协议，让 AI 工具能像调用函数一样访问数据库、API 等外部数据源 <VibeExample id="3.4" />

::left::

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- 在项目根目录创建 `.cursor/mcp.json` 配置 [MCP Servers](https://cursor.com/docs/context/mcp)
- 通过 `@server-name` 在对话中直接调用外部服务

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": [
        "-y", 
        "@modelcontextprotocol/server-postgres"
      ],
      "env": { "DATABASE_URL": "${env:DATABASE_URL}" }
    }
  }
}
```

::right::

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- 在 `~/.claude/settings.json` 全局或项目 `.claude/mcp.json` 本地配置 [MCP Servers](https://code.claude.com/docs/en/build-with-claude-code/configure-mcp-servers)
- Claude Code 会自动加载配置的 MCP 服务并在对话中提供调用

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y", 
        "@modelcontextprotocol/server-filesystem", 
        "/path/to/allowed"
      ]
    }
  }
}
```

[➔ 参阅官方标准指南](https://modelcontextprotocol.io/docs/getting-started/intro)

# 生命周期钩子 (Hooks)

> 在 AI 执行关键操作前（如提交代码）插入验证逻辑。测试不通过则自动阻断提交并通知 AI 修复 <VibeExample id="3.5" />

::left::

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- 在 `.cursor/hooks/` 目录配置 [Hooks](https://cursor.com/docs/agent/hooks)，支持文件操作生命周期拦截（如 PostEdit、PreCommit 等）

```json
{
  "version": 1,
  "hooks": {
    "stop": [
      {
        "command": ".cursor/hooks/notify-on-stop.sh"
      }
    ]
  }
}
```

::right::

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- 支持在 `.claude/hooks/` 目录配置 JSON Hooks 或使用 [Prompt Hooks](https://code.claude.com/docs/en/hooks)

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/notify-on-stop.sh"
          }
        ]
      }
    ]
  }
}
```

# 多重分身 (Sub Agents)

> 复杂任务由多个子 Agent 协作完成。总指挥负责任务拆解和验收，各子 Agent 专注自己的领域 <VibeExample id="3.6" />

::left::

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- 通过 [Subagents](https://cursor.com/docs/context/subagents) 功能创建专用子 Agent
- AI 在执行复杂任务时，会自动调用相应的子 Agent 执行特定子任务（如读日志、搜文档）

::right::

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- 在 `.claude/agents/*.md` 目录定义 [Sub-Agents](https://code.claude.com/docs/en/sub-agents)，主 Agent 可调用这些子 Agent
- 支持通过 YAML Frontmatter 定义 Agent 名称、描述、模型和可用工具

```markdown
---
name: frontend-agent
description: 前端开发 Agent
model: sonnet
allowed-tools: Read, Write, Edit, Bash, Glob
---
你是一个专业的前端开发者，负责根据需求编写高质量的 React/Vue 组件。
```

# Daily Usage 日常使用

> 日常高频使用的基础功能，提升交互效率

| 功能 | Cursor | Claude Code |
|------|--------|-------------|
| @ 追加上下文 | `@file`、`@folder`、`@docs`、`@symbols` 多种引用类型 | `@filename` 文件引用，支持路径自动补全 |
| / 斜杠命令 | `/chat` 开启对话、`/edit` 编辑模式、`/generate` 生成代码 | `/edit` 编辑模式、`/commit` 提交代码、`/pr` 创建 PR、`/test` 运行测试 |
| Session 会话 | 对话历史保持上下文，关闭后可继续上次对话 | `/resume` 恢复会话、`/rename` 重命名会话、[会话持久化](https://code.claude.com/docs/en/persistence) |
| Modes | Agent / Ask / Plan / Debug 模式切换，适应编码、问答、探索、排错不同场景 | 支持 `--permission-mode plan` 方案探索，以及交互/命令行模式切换 |

# 杀手特性

> 每个工具的独占能力，形成差异化竞争优势

::left::

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- **[BugBot](https://cursor.com/docs/bugbot)**：运行时调试 + 堆栈分析，当代码运行抛错或 UI 崩溃，点击 Bug 图标将堆栈路径、变更文件、终端日志整合为完整上下文

::right::

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- **[Memory](https://code.claude.com/docs/en/memory)**：跨会话记忆，长期维持项目记忆，记住微小决策与偏好
- **[Agents Team](https://code.claude.com/docs/en/agent-teams)**：多 Agent 协作，复杂任务由多个子 Agent 协作完成
- **[Worktrees](https://code.claude.com/docs/en/common-workflows#worktrees)**：避免上下文污染的并行开发隔离工作流
- **[Git Integration](https://code.claude.com/docs/en/common-workflows#pull-requests)**：完整的自动化提交与 PR 创建开发闭环
- **[Headless Mode](https://code.claude.com/docs/en/cli-reference#headless)**：自动化批处理与无缝衔接 CI/CD 集成
