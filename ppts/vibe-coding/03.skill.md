# 场景 1: 修复 Bug

**目标**：快速定位并修复回归或逻辑错误

**角色分工**：

- Driver（代码作者）
- Navigator（审查者）
- AI 助手（建议/补全）

**示例 Prompt**：

```
我在 src/utils/calc.js 中发现测试失败
错误为：TypeError: undefined
相关函数如下：<粘贴代码>
请帮我定位可能的原因并给出修复建议。
```

::right::

<div style="padding-left: 20px;">

# 场景 2: 编写单元测试

**目标**：为关键函数补全测试与边界用例

**示例 Prompt**：

```
为下列函数生成 Jest 测试用例，
覆盖：
- 正常输入
- 异常输入
- 边界条件

代码：
<粘贴函数代码>

输出格式：返回可直接复制到
__tests__/ 的完整测试文件。
```

</div>

---


# 场景 3: 重构与优化

**目标**：提高可读性、减少重复、提升性能

**示例 Prompt**：

```
背景：需要重构以下函数/模块
目标：提升可读性/消除重复/提升性能

代码：<粘贴代码>

要求：
- 给出替代实现并说明为什么更优
- 如果有 breaking change 请指出
```

::right::

<div style="padding-left: 20px;">

# 场景 4: 代码审查 (PR)

**目标**：快速得到可操作的审查要点

**示例 Prompt**：

```
请作为资深工程师审查以下 PR
列出高优先级问题并给出逐条修复建议

检查维度：
- 安全性
- 性能
- 边界条件
- 可维护性

PR 摘要：<简要描述>
重要文件：<paths>
```

</div>

---

# 场景 5: 文档与注释生成

**目标**：将复杂实现转为可读文档，提高可维护性

**示例 Prompt**：

```
请为以下模块生成 README 概要，包含：
- 功能描述
- 参数说明
- 示例用法
- 注意事项

代码：<粘贴模块主文件或导出函数列表>
输出格式：Markdown，包含示例代码块
```
