# 常见问答与最佳实践

---


## ❓ 常见问题

<div style="margin-top: 10px;">

**Q: AI 生成的代码可以直接合并吗？**

A: 不建议。应先通过：

- ✅ 本地运行测试
- ✅ 代码审查
- ✅ 安全检查

**Q: 如何保护隐私与敏感信息？**

A: 采取以下措施：

- 避免在 prompt 中包含密钥、用户数据
- 对外部服务启用最小化传输
- 必要时使用本地 LLM（如 Ollama）

</div>

::right::

<div style="padding-left: 20px; margin-top: 10px;">

**Q: 当 AI 给出错误建议怎么办？**

A: 请记住：

- 把 AI 作为 "建议者" 而非 "裁决者"
- 将其产出作为候选解
- 始终由开发者验证和决策

**Q: 如何提高 AI 输出质量？**

A: 遵循 3S 原则：

- **Simple** - 简洁清晰
- **Specific** - 明确具体
- **Short** - 精炼扼要

</div>

# ✨ 最佳实践总结

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 24px; margin-top: 20px;">

<div style="padding: 20px; background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 12px; color: white;">

### 📚 SOP (标准作业程序)

1. **定义 (Define)**: 明确输入与输出格式
2. **上下文 (Context)**: 使用 @Files/@Topics 聚焦
3. **生成 (Generate)**: 使用 Composer 或 CLI
4. **验证 (Verify)**: 运行测试或人工审查
5. **迭代 (Iterate)**: 基于反馈微调 Prompt

</div>

<div style="padding: 20px; background: linear-gradient(135deg, #11998e 0%, #38ef7d 100%); border-radius: 12px; color: white;">

### 🔍 PR 审查清单

在 PR 流程加入 AI 生成代码专项检查：

- ☐ 测试覆盖 (AI 代码必须有测试)
- ☐ 性能影响 (避免 O(n^2))
- ☐ 边界条件 (null/undefined)
- ☐ 安全隐患 (SQL 注入/密钥)

</div>

<div style="padding: 20px; background: linear-gradient(135deg, #fc5c7d 0%, #6a82fb 100%); border-radius: 12px; color: white;">

### 📊 持续改进

- 记录高效 prompts 与策略
- 收集团队反馈
- 识别常见问题模式
- 迭代优化工作流 (.cursorrules)

</div>

<div style="padding: 20px; background: linear-gradient(135deg, #f093fb 0%, #f5576c 100%); border-radius: 12px; color: white;">

### 🛠️ 工具配置

- IDE 插件正确配置
- 合理设置隐私策略
- 配置使用限额
- 启用审计日志

</div>
