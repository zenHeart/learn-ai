# 4.3 疑难 Bug 定位与修复

## 概念解析
在复杂的业务系统中，出了 Bug 后只看日志常常无法立刻定位，因为调用链路通常横跨 Controller、Service、Database 和各种 Utils 层。

在结合了 Cursor / Claude Code 的场景下，解决线上疑难杂症的标准 SOP 变成了：
1. **喂给日志**：将 Sentry 或者 Kibana 的一长串复杂报错和堆栈扔给 AI。
2. **AI 全局检索**：终端型的 Agent 会顺着堆栈中的文件名，自己去本地项目里跑 `grep` / 文件读取，把发生错误的方法以及它依赖的上下游代码全找出来。
3. **出具处方**：分析参数传递，找出为什么会出现 `null pointer`，并直接把修复后的完美 Diff (如可选链补全、防空判断) 输出。

## 运行示例

进入根目录（`ppts/vibe-coding/examples`），运行：
```bash
npm run demo:4.3
```

## 核心要点
* 大模型长于从成百上千行的“噪音”中敏锐捕捉关键线索（大海捞针的能力极强）。
* Agent 通过自动寻址读取相关代码文件，省去了开发者反复切文件、搜索的时间。
