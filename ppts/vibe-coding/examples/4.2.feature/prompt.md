# 4.2 演示指南：验证驱动开发 (VDD)

## 操作步骤

1. 打开终端，运行以下命令：
   ```bash
   npm run demo:4.2
   ```
2. 你会看到终端输出了红色的报错信息（由于没有实现 `trailing` 逻辑和 `this` 绑定，测试 2 和 3 过不去）。
3. **不要自己修代码**，在终端中唤起 Claude Code (输入 `claude`)，或者在 Cursor 的 Chat 侧边栏里，粘贴整个报错堆栈，并且附加一句简单的 Prompt：
   > `修复 4.2.vdd/throttle.ts 里的 bug 以让测试通过。`
4. **观察结果**：
   - AI 完全能看得懂 Node 测试运行器的报错堆栈。
   - AI 会利用堆栈信息，精准定位到是 `trailing` 参数没处理和 `this` 丢失的问题，然后生成并修改代码。
5. 再次运行 `npm run demo:4.2`，你会看到所有测试变成了绿色的 ✅。这就是 VDD（验证驱动开发）的强力闭环！
