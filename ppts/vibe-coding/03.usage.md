# 项目上下文基建 (AGENTS.md)

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 20px; margin-top: 20px;">

<div>

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- **原生阵地**：支持全局解析，常放置在项目根目录 [AGENTS.md](https://cursor.com/docs/context/rules#agentsmd) 或 `.cursor/rules` 文件夹下的规则文件。
- **功效**：设定编程规约基调，是每次会话注入不可或缺的隐形 Prompt。

<VibeExample id="3.1" />

```markdown
# 架构与规范
- 统一使用 TypeScript 严格模式
- 必须补齐 React PropTypes
- 单测覆盖率必须达到 80% 才能提交
```

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- **记忆体挂载**：使用专有的 [CLAUDE.md](https://code.claude.com/docs/en/best-practices#write-an-effective-claude-md)（或通过 `/init` 指令交互式创建）。
- **功效**：相较被动加载，终端 Agent 在冷启动会主动阅读并自我校验。

```bash
# CLI 独享指令边界
$ claude -p "Review my changes against CLAUDE.md"
```

</div>

</div>

# 细粒度模块管控 (Rules)

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 20px; margin-top: 20px;">

<div>

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- **精准制导**：在 `.cursor/rules/*.mdc` 目录管理 [Project Rules](https://cursor.com/docs/context/rules#project-rules)。
- **黑魔法**：利用 YAML Frontmatter 设定匹配目标通配符（Globs），按需触发。

<VibeExample id="3.2" />

```yaml
---
description: React 组件开发守则
globs: src/components/**/*.tsx
alwaysApply: false
---
只能使用函数式组件 (Hooks)，禁止编写 Class。
```

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- **模块化法则**：存放在 `.claude/rules/` 文件夹下的独立 [Modular Rules](https://code.claude.com/docs/en/memory#modular-rules-with-claude%2Frules%2F)。
- **动态寻址**：无需手动匹配路径，Agent 在处理不同模块时具备高度自主寻址能力。

```markdown
# 最佳实践
当你在 `tests/` 下编写用例时，请默认使用
Vitest 框架，而非 Jest。
```

</div>

</div>

# 工作流沉淀 (Commands & Skills)

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 20px; margin-top: 20px;">

<div>

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- **自定义捷径**：通过 [Commands](https://cursor.com/docs/context/commands) 斜杠快速呼出日常高频咒语。
- **本地扩展**：兼容 [Skills](https://cursor.com/docs/context/skills)，供 AI 调度的原生本地动作函数库。

```json
{
  "commands": {
    "review": "审查当前文件安全性"
  }
}
```

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- **自动化黑盒**：在 `~/.claude/skills/*.md` 编写带有前后台钩子的 [专属 Skills](https://code.claude.com/docs/en/skills) 。
- **权限缩圈**：能够极限掌控 Agent，例如运行审查技能时剥夺其代码篡改权限。

```yaml
---
name: security-audit
description: 代码安全扫描
allowed-tools: Read, Grep
---
请扫描依赖树，全量报告 CVE 漏洞。
```

</div>

</div>

# 外部能力接入 (MCP)

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 20px; margin-top: 20px;">

<div style="grid-column: span 2;">

**Model Context Protocol (MCP)**：一种让大模型标准访问本地或远端数据源的开源连通协议。
[➔ 参阅官方标准指南](https://modelcontextprotocol.io/docs/getting-started/intro)

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- **即插即用**：设置面板一键填入 [MCP 端口](https://cursor.com/docs/context/mcp)。
- **实景演练**：
  聊天框输入：“@postgres 库里有哪些用户的积分高于100？” 
  无需编写任何后接代码，IDE 帮你直通数据库。

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- **无头驱动**：在底层的 `settings.json` [静默挂载 Server](https://code.claude.com/docs/en/build-with-claude-code/configure-mcp-servers)。
- **实景演练**：
  “分析 /docs 下的产品设计稿，并去 Jira 里核对 PRD 和测试报告，如果一致，拉取远程最新分支直接产出脚手架。”

</div>

</div>

# 生命周期钩子 (Hooks)

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 20px; margin-top: 20px;">

<div>

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- **文件屏障**：目前尚未支持通用全生命周期 Hook。
- **替代方案**：主要依靠开启或关闭 AI 修改确切文件的权限，或者通过 Rules 阻断特定模式的前置生成。

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- **原生体系**：支持前置/后置 [Hooks](https://code.claude.com/docs/en/hooks) 脚本拦截 (`.claude/hooks`)。
- **实景演练**：
  编写 `pre-commit` 钩子：每当 Agent 执行完了所有工作准备 Git 提交时，如果检测到没有跑通 `npm run lint`，强制剥夺其提交权限并挂起。

```bash
# pre-commit.sh
if ! npm run lint; then
  echo "Lint failed. Claude Code operation halted."
  exit 1 
fi
```

</div>

</div>

# 多重分身 (Sub Agents)

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 20px; margin-top: 20px;">

<div>

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> IDE 实现 (Cursor)</h3>

- **调度黑盒**：在底层封装了 [Agent 模式](https://cursor.com/docs/context/subagents)。
- **表现形式**：当打开 `Composer` 时，AI 遇到复杂需求，会自动分裂出专门去终端读 log 的小兵，或者去搜文档的小兵，用户仅查看汇总。缺乏人为显式组织结构。

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> CLI 实现 (Claude Code)</h3>

- **显式编排**：原生支持通过配置定义 [Sub-Agents](https://code.claude.com/docs/en/sub-agents)。
- **架构演进**：开发者能以树状拓扑硬编码派生不同职能的子智能体。
  例如：“主节点分配任务” -> “前端子 Agent 写 UI” / “后端子 Agent 建库”。

</div>

# 杀手特性

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 20px; margin-top: 20px;">

<div>

<h3 class="flex items-center gap-2"><img src="/cursor.svg" class="w-6 h-6" alt="Cursor" /> Cursor 独占：原力审视 (Debugger)</h3>

- **全息断层扫描**：当代码运行抛错，或者 UI 崩塌，点一下“Bug”图标，Cursor 会将**堆栈调用路径、最近变更过哪些文件、终端报错日志**全部揉进一个上下文，直接吐出多文件协同病历单。
- 为什么强？它打通了 IDE 编辑态与运行时态的隔离墙。

</div>

<div>

<h3 class="flex items-center gap-2"><img src="/claude.svg" class="w-6 h-6" alt="Claude Code" /> Claude 独占：记忆迷宫 (Context Memory)</h3>

- **超级海马体**：长期维持庞大项目的跨会话记忆，不仅记住你写过什么代码，还能记住你[此前的微小决策与偏好](https://code.claude.com/docs/en/memory)。
- 为什么强？即使你昨天关了电脑，今天跑一句 `继续昨天那个遗留的支付重构`，终端立刻通过向量回溯前置语境，完全不需要再手动拖拽引用文件！

</div>

</div>
