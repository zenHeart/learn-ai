# 01. User Prompt: 意图注入

---
layout: vibe-step
activeStep: start
---

### 核心概念：**Token (信息的最小单位)**

当你在编辑器中输入自然语言或选中代码时，AI 看到的是一组 **Tokens**。

- **词元化 (Tokenization)**：代码比自然语言更适合 AI，因为其语法结构使得 Token 边界更清晰。
- **意图捕获**：不仅是你说的话，你的**光标位置**、**当前选中的代码块**、**正在报错的堆栈**，都是被注入的隐式 Prompt。

> [!TIP]
> **Vibe Coding 智慧**：提示词的清晰度直接决定了 Token 预测的准确性。

# 03. LLM Reasoning: 推理与规划

### 核心技术：**Transformer 与预测模型**

大模型并不是在“思考”，它是在做**概率预测**。

- **Transformer 架构**：通过“注意力机制”理解长距离的代码依赖关系（例如：知道 100 行外的变量是在哪里定义的）。
- **自回归生成**：预测下一个最可能的代码 Token。
- **思维规划 (Planning)**：生成代码前，Agent 会先输出一段“思维链（CoT）”，规划要修改哪些文件。

```typescript
// 推理过程伪代码
输入: "给 Button 增加 loading 状态"
规划: 1. 修改 types.ts 增加属性; 2. 修改 Button.tsx 实现逻辑; 3. 更新 index.css 动画。
```

# 04. Take Action: 执行行动

### 核心能力：**工具调用 (Tool Use / Function Calling)**

这是 Vibe Coding 从“说”到“做”的关键跨越。

| 工具 | 对应操作 | 意义 |
|:---|:---|:---|
| **WriteFile** | 直接修改磁盘上的代码 | 无需开发者进行复制粘贴 |
| **Terminal** | 运行 `npm install` 或 `git commit` | 掌控开发环境的执行权 |
| **LSP** | 请求 IDE 获得语法提示 | 获取精准的代码元数据 |

# 05. Verify & Feedback: 自动验证与循环

### 核心概念：**Agentic Loop (闭环)**

**Agent 不会写完就跑。**

1. **自动验证 (Verify)**：代码写入后，Agent 自动调用 `Linter` 或检查编译错误。
2. **错误回流 (Feedback Loop)**：如果报错，报错信息作为**新 Context**，自动触发下一轮推理。
3. **自我修复**：
   - AI: “哦，我刚才漏掉了一个 Import。”
   - AI: “自动重试，修正代码。”

# 06. Human In The Loop: 人机协作

### 核心理念：**打断与转向 (Steer)**

AI 的预测不是 100% 准确的。Vibe Coding 强调的是**人类的实时感官与导航**：

- **及早打断**：发现 AI 推理偏离（例如用了错误的组件库），点击 **Interrupt**。
- **重定向**：注入新的约束条件 —— “不用这个库，改用原生 CSS 实现”。
- **最终把关**：AI 完成后，开发者仅需对增量（Diff）进行最后的 Code Review。
