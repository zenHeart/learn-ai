# 实战场景 1：新项目冷启动 (Intent to Architecture)

> 从 0 → 1 构建项目：输入是**模糊意图**（脑暴想法 / 竞品参考），输出是**可运行的项目**。<VibeExample id="4.1" />


<div class="pair-grid">
  <div class="pair-card blue">
    <div class="card-title">🗺️ Plan（规划）</div>
    <div class="card-meta">输入：意图描述</div>
    <div class="card-body">用 **Plan Mode** 驱动 AI 生成 `AGENTS.md`（技术栈决策）和架构草图，明确模块边界与目录结构。</div>
    <div class="card-out">产出：user-guide.md</div>
  </div>
  <div class="pair-card purple">
    <div class="card-title">🔍 Assess（对齐）</div>
    <div class="card-meta">评估：user-guide.md</div>
    <div class="card-body">检查关键约束是否遗漏（如技术栈选型等）。 MVP 版本是否符合预期。有偏差 → 返回 Plan 补充。</div>
    <div class="card-out">产出：design.md + todo.md</div>
  </div>
  <div class="pair-card green">
    <div class="card-title">⚡ Implement（执行）</div>
    <div class="card-meta">粒度：每次 = 一个模块</div>
    <div class="card-body">按 Todo 逐模块执行： 按照设计架构，可以多 Agents 并行执行，先完成 MVP, 在迭代优化。</div>
    <div class="card-out">产出：可运行的 MVP</div>
  </div>
  <div class="pair-card amber">
    <div class="card-title">✅ Review（验收）</div>
    <div class="card-meta">卡点：每模块完成后</div>
    <div class="card-body">通过测试，人工验证，对照 blueprint.md 检查目录结构；架构偏差 → 回溯 Plan。</div>
    <div class="card-out">最终：可运行项目</div>
  </div>
</div>

# 实战场景 2：feature 需求开发 (PRD to PR)

> 已有项目新增功能：输入是 **PRD + 设计稿**，输出是经过测试验证的**功能 PR**。<VibeExample id="4.2" />

<div class="pair-grid">
  <div class="pair-card blue">
    <div class="card-title">🗺️ Plan（规划）</div>
    <div class="card-meta">输入：PRD.md + UI 设计稿</div>
    <div class="card-body">@ 引用 PRD.md 和 UI 截图或者 figma MCP，用 **Plan Mode** 生成 `design.md`（接口/数据流/组件边界）和 `todo.md`（Story 拆解）。</div>
    <div class="card-out">产出：design.md + todo.md</div>
  </div>
  <div class="pair-card purple">
    <div class="card-title">🔍 Assess（对齐）</div>
    <div class="card-meta">评估：design.md</div>
    <div class="card-body">Review 接口字段完整性、数据流向、组件归属；确认 todo 粒度可执行。方案不合理 → 返回 Plan 补充。</div>
    <div class="card-out">产出：高质量 design.md</div>
  </div>
  <div class="pair-card green">
    <div class="card-title">⚡ Implement（执行）</div>
    <div class="card-meta">粒度：每次 = 一个功能</div>
    <div class="card-body">按 `todo.md` 逐条执行，@ 引用 `AGENTS.md` 约束产出规范。复杂 Story 可嵌套子 PAIR 循环。</div>
    <div class="card-out">产出：增量功能+测试用例</div>
  </div>
  <div class="pair-card amber">
    <div class="card-title">✅ Review（验收）</div>
    <div class="card-meta">卡点：每个 Story 完成后</div>
    <div class="card-body">功能测试通过；对比 Diff 关注边缘情况；逻辑缺陷 → 回溯 Plan 修正 design.md。</div>
    <div class="card-out">最终：功能 PR</div>
  </div>
</div>

# 实战场景 3：hotfix Bug 修复 (Crash to PR)

> 从线上问题到闭环修复：输入是 **Crash 堆栈 / 日志**，输出是带 Repro Test 的**修复 PR**。 <VibeExample id="4.3" />

<div class="pair-grid">
  <div class="pair-card blue">
    <div class="card-title">🗺️ Plan（归因）</div>
    <div class="card-meta">输入：Crash 堆栈 / Sentry / 业务日志</div>
    <div class="card-body">注入堆栈 + 日志，让 AI 跨文件追踪调用链，生成 `root-cause.md`（根因分析）和 `fix-todo.md`（修复计划）。</div>
    <div class="card-out">产出：design.md</div>
  </div>
  <div class="pair-card purple">
    <div class="card-title">🔍 Assess（评估）</div>
    <div class="card-meta">评估：design.md </div>
    <div class="card-body">确认根因是否准确；修复面是否最小化（避免牵连无关模块）；归因存疑 → 回流 Plan 追加日志范围。</div>
    <div class="card-out">产出：todo.md</div>
  </div>
  <div class="pair-card green">
    <div class="card-title">⚡ Implement（修复）</div>
    <div class="card-meta">粒度：每次 = 一个用例</div>
    <div class="card-body">先写复现测试（固化 Bug），再修复代码，确保测试由红变绿。按 `fix-todo.md` 原子化执行。</div>
    <div class="card-out">产出：修复 + test</div>
  </div>
  <div class="pair-card amber">
    <div class="card-title">✅ Review（验收）</div>
    <div class="card-meta">卡点：每处修复完成后</div>
    <div class="card-body">Repro Test 通过；无新 Warning；全量测试无 regression；PR 描述含复现步骤与根因说明。</div>
    <div class="card-out">最终：修复 PR</div>
  </div>
</div>

<div class="summary-grid">

<div class="summary-card">

📚 PAIR 工作流 SOP

1. **Plan (计划)** 明确需求，制定详细计划
2. **Assess (评估)** 进入 Plan Mode 确认 AI 理解
3. **Implementation (实施)** 标准化执行，Agent Loop
4. **Review (评审)** 人工评审兜底，边界/性能/安全审查

</div>

<div class="summary-card">

🔄 双微循环机制

- **理解对齐循环** (Plan ↔ Assess): 迭代直到 AI 完全理解
- **Checkpoint 验证循环** (Implementation ↔ Review): 每个小步骤后的验证
- **回退路径**: 理解偏差 → Plan / 需精细化 → Plan

</div>

<div class="summary-card">

🧠 Vibe 核心：从 Builder 到 Reviewer

1. **意图第一** 代码只是意图的副产品，关注"你想要什么"。
2. **审查为先** 你的价值不在于打字快慢，而在于识别"正确的代码"。
3. **拒绝手动** 只要能通过规则/测试约束 AI 的，绝不自己动手改代码。

</div>

<div class="summary-card">

📊 组织与持续改进

- 通过 `AGENTS.md` 管理项目和通用规范
- 利用 commands/rules 沉淀团队 prompt
- 利用 SKILL 沉淀各种工作流
- 利用 MCP 提供大模型内网信息

</div>
