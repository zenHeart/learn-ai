# Vibe Coding PPT 与示例对齐优化建议

## 目标对齐基线
- 课程目标：正确理解 Vibe Coding、高效利用相关工具、解决实际问题（01.overview.md）。
- 会话目标：理解理念与协作方式、熟悉工具最佳实践、在真实开发场景中落地（README.md）。

## 当前覆盖与匹配情况（PPT ↔ 示例）
- 原理类概念（Tokenizer/Embedding/Attention/Transformer/Tool Use）
  - PPT：02.principle.md
  - 示例：2.1-tokenizer、2.2-embedding、2.3-attention、2.4-tool-use
  - 结论：概念匹配度高，具备可运行演示基础。
- 使用与体系（工具、AGENTS.md、Rules、Commands/Skills、MCP、Hooks、Sub-Agents）
  - PPT：03.usage.md
  - 示例：3.1-agents-md、3.2-rules-matching、3.3-commands-skills、3.4-mcp-protocol、3.5-hooks、3.6-sub-agents
  - 结论：结构覆盖完整，但存在不可运行示例与互动性不足问题。
- 实战场景（冷启动、VDD、疑难 Bug 修复）
  - PPT：04.practice.md
  - 示例：4.1-cold-start、4.2-vdd、4.3-bug-fix
  - 结论：主题对应正确，但 VDD 演示入口有运行问题，Bug 修复偏“脚本讲述”而非“真实排查”。

## 主要问题与修改必要性
- 示例与讲稿之间的可复现性不足
  - 3.1 示例存在错误导入导致不可运行。
  - 4.2 示例脚本入口无法执行，演示链路中断。
- 课程大纲包含 Context Window / Memory / Agent Loop，但示例缺失或仅为静态说明。
- 使用与实战部分偏日志模拟，缺少可让听众亲手体验的真实工作流。

## 优化方向与理由（含目的与收益）

### 1) 修复不可运行的示例，保证演示稳定性
- 修改原因：影响“解决实际问题”的演示可信度。
- 目的：保证现场与复现流程顺畅。
- 收益：增强讲者控制力与听众信任。
- 具体调整：
  - 修正 3.1 示例的错误导入，确保演示可启动。
  - 修正 4.2 测试入口的导入路径或构建流程。

### 2) 将 3.x/4.x 从“日志模拟”提升为“可操作演练”
- 修改原因：现有演示缺乏真实 IDE 交互体验。
- 目的：让听众在 Cursor/Claude Code 中完整跑通“指令 → 产出 → 验证”的闭环。
- 收益：满足“高效利用工具”与“真实场景落地”的目标。
- 具体调整：
  - 3.1/3.2：提供真实 AGENTS.md 与 Rules 文件，示例代码保持残缺，让 AI 在编辑器内补全。
  - 4.2：保留真实测试用例与失败实现，让听众依赖 AI 修复实现。
  - 4.3：构造多文件场景与真实日志输入，演示跨文件定位与修复。

### 3) 补齐 Context Window / Memory / Agent Loop 的可运行示例
- 修改原因：大纲明确为核心原理，但缺少可复现示例。
- 目的：从“概念理解”升级为“可动手验证”。
- 收益：强化“正确理解 Vibe Coding”的目标。
- 具体调整：
  - Context Window：构造超长上下文输入与截断示例，展示信息丢失与摘要策略。
  - Memory：模拟会话记忆与长期偏好恢复的流程。
  - Agent Loop：用最小闭环脚本体现“计划 → 执行 → 验证 → 修复”。

### 4) 在 PPT 中添加示例入口提示
- 修改原因：听众难以在 PPT 与示例之间建立路径。
- 目的：统一观众心智路径“看到 → 运行 → 理解”。
- 收益：提高吸收效率与现场跟练成功率。
- 具体调整：
  - 每个关键概念页加一行 Demo 入口提示（npm run demo:X.Y）。

### 5) 统一 Prompt 指令卡与操作指南
- 修改原因：工具使用方式分散且不易复用。
- 目的：让听众在不同工具中快速迁移。
- 收益：提升工具上手速度，降低讲解成本。
- 具体调整：
  - 为每个示例提供简短的 Prompt 指令模板与使用说明。

## 最终优化建议（面向目标达成）
- 优先修复运行问题，确保演示链路可复现。
- 将使用/实战示例从“讲故事”升级为“可操作实战”。
- 补齐 Context Window / Memory / Agent Loop 的最小可运行示例。
- PPT 与示例建立明确的 Demo 映射指引。
- 为每个实战场景提供统一 Prompt 指令卡，降低学习门槛。

## 预期结果
- PPT 与示例完全对应，概念可演示、可运行、可复现。
- 听众能在现场完成真实工具操作，形成“可落地”的 Vibe Coding 认知。
- 满足 README 与 01.overview.md 的全部目标要求。
