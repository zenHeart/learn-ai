# Prompt 工程与快速交互

- 为什么需要明确的 prompt：帮助 AI 快速定位问题域、约束输出格式、提高可复现性
- Prompt 结构建议：\n  1) 背景（场景 + 目标）\n  2) 代码上下文（文件/函数/错误信息）\n  3) 期望输出（实现/修复/测试/解释）\n  4) 限制条件（语言、风格、接口）

- 示例模板（修复 Bug）:

```
背景：项目使用 Node.js + Jest，当前测试 `utils/calc.js` 失败，报错信息：<错误信息>
上下文：相关函数如下：<粘贴函数代码>
期望：请定位可能原因并给出修复补丁，要求提供修改后的完整函数代码与简要解释。
```

- 示例模板（生成测试）:

```
说明：为下列函数生成 Jest 测试用例，覆盖正常输入、异常输入与边界条件。
代码：<函数代码>
输出格式：返回一个可直接复制到 `__tests__` 的代码块。
```
