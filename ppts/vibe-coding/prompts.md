# Vibe Coding - 常用 Prompt 模板

下面模板均为示例，使用时替换 `<...>` 部分并根据团队规范调整细节。

---

## 1. 修复 Bug（定位与补丁）

```
背景：项目为 Node.js/TypeScript，测试/运行中出现错误：<简要错误信息>
相关文件：<path/to/file.js>
相关代码：
<粘贴最小可复现的函数或代码片段>

任务：请定位可能的原因并返回一个能修复问题的补丁。

要求：
- 给出修改后的完整函数实现
- 简要说明修改原因与潜在风险
```

---

## 2. 生成 Jest 单元测试

```
说明：为下列函数生成 Jest 测试用例，覆盖以下场景：
- 正常输入
- 异常输入
- 边界条件

代码：
<粘贴函数代码>

输出格式：返回一个可直接复制到 `__tests__/` 的完整测试文件内容。
```

---

## 3. 重构建议

```
背景：下面是需要重构的函数或模块
目标：提升可读性 / 消除重复 / 提升性能

代码：
<粘贴代码>

要求：
- 给出替代实现并说明为什么更优
- 如果有 breaking change 请指出影响范围
- 保持接口兼容性
```

---

## 4. PR 审查要点

```
请作为资深工程师审查以下 PR，列出高优先级问题并给出逐条修复建议。

检查维度：
- 安全性
- 性能
- 边界条件
- 可维护性

PR 摘要：<简要描述>
重要文件/函数：<paths>

输出格式：使用 Markdown 列表，按优先级排序
```

---

## 5. 文档生成（README/注释）

```
请为以下模块生成 README 概要：

代码：
<粘贴模块主文件或导出函数列表>

包含：
- 功能描述
- 参数说明
- 示例用法
- 注意事项

输出格式：Markdown，包含示例代码块
```

---

## 6. 代码解释

```
请解释这段代码做了什么：

代码：
<粘贴代码>

回答以下问题：
1. 核心逻辑是什么？
2. 时间/空间复杂度如何？
3. 可能的风险或边界情况有哪些？
```

---

## 7. 性能优化

```
请分析以下代码的性能瓶颈并提供优化建议：

代码：
<粘贴代码>

场景：<描述使用场景，如：大数据量、高并发等>

要求：
- 列出可能的性能问题
- 给出优化后的实现
- 说明优化的原理与预期效果
```

---

## 8. API 设计建议

```
我需要设计一个 API 来实现 <功能描述>

约束条件：
- 语言/框架：<如 Node.js + Express>
- 调用方式：<REST / GraphQL / RPC>
- 用户场景：<描述>

请提供：
1. API 接口设计（路径、方法、参数）
2. 返回值结构
3. 错误码定义
4. 简单的实现示例
```

## 9. 数据库查询优化

```
请优化以下 SQL/数据库查询：

原始查询：
<粘贴查询>

表结构：
<描述相关表结构或索引>

问题：<查询慢 / 资源消耗高 / 等>

请提供优化后的查询和解释
```

## 10. 代码迁移/升级

```
我需要将以下代码从 <旧版本/框架> 迁移到 <新版本/框架>：

原始代码：
<粘贴代码>

要求：
- 保持功能一致
- 使用新版本的最佳实践
- 列出需要注意的 breaking changes
```

## 使用说明

1. **选择合适的模板**：根据当前任务类型选择对应模板
2. **填充具体内容**：替换 `<...>` 部分为实际代码和描述
3. **迭代优化**：根据 AI 返回结果，可能需要追问细化
4. **验证输出**：始终验证 AI 生成的代码（运行测试、人工审查）
5. **收录高效 prompt**：将效果好的 prompt 变体收录到此文件

---

## Prompt 编写原则（3S 原则）

- **Simple（简洁）**：使用清晰的语言，避免歧义
- **Specific（具体）**：提供足够的上下文和约束条件
- **Short（精炼）**：保持适当长度，避免冗余信息
