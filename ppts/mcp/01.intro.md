# 1. 为什么需要 MCP？(溯源痛点)

> 在 MCP 出现之前，AI 访问外部数据面临着灾难性的 **N × M 集成噩梦**。

::left::

<div class="mt-4">

### 💥 过去的痛点 (N × M)
假设市场上有 **N** 个优秀的 AI 工具（Cursor, Claude, Windsurf, v0 等）。
你的企业有 **M** 个数据源（本地数据库, Jira, Gitlab, 飞书文档等）。
- 每个 AI 工具都需要**独立适配**这些数据源的 API。
- 需要编写 `N × M` 次集成代码！
- 极难维护，且 API 密钥四处散落，存在严重**安全风险**。

</div>

::right::

<div class="mt-4">

### 💡 Anthropic 的解法 (N + M)
2024年底，Anthropic 提出了 [Model Context Protocol (MCP)](https://www.anthropic.com/news/model-context-protocol)。
- 制定了一个**通用的双向通信标准**。
- 将架构变为 `N + M`。
- **只需开发一次 MCP Server** 暴露你的私有数据，所有的 AI 客户端（只要支持 MCP 协议）都能**即插即用**地直接读取。

</div>

# MCP 是什么？

> **MCP (Model Context Protocol)** 就像是数字世界的 **USB-C 接口标准**。

以前，给不同的手机充电需要准备 Micro-USB、Lightning、专有接口等一堆不同的线。
现在，一根 **USB-C (MCP)** 解决所有通信问题，让 AI 应用能够安全、标准化地与外部系统（甚至本地电脑的文件系统）无缝连接。
