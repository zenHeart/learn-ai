# 实战案例

## 案例 1：自动化代码审查

**场景**: 团队需要标准化的代码审查流程

**解决方案**: 创建 `code-review` Skill

```yaml
name: code-review
description: 标准化代码审查流程
trigger: /review

steps:
  - name: 获取变更
    action: git_diff

  - name: 运行 linter
    action: run_command
    params:
      command: npm run lint

  - name: 运行测试
    action: run_command
    params:
      command: npm test

  - name: 生成审查报告
    action: format_review
```

## 案例 2：自动化部署

**场景**: 简化部署流程

**解决方案**: 创建 `deploy` Skill

```yaml
name: deploy
description: 自动化部署流程
trigger: /deploy

steps:
  - name: 检查分支
    action: check_branch

  - name: 运行测试
    action: run_command
    params:
      command: npm test

  - name: 构建
    action: run_command
    params:
      command: npm run build

  - name: 部署
    action: run_command
    params:
      command: npm run deploy
```

---

## 案例 3：标准化 PR 创建

**场景**: 统一 PR 格式和流程

**解决方案**: 创建 `pr-create` Skill

```yaml
name: pr-create
description: 创建标准化 Pull Request
trigger: /pr

steps:
  - name: 检查分支状态
    action: git_status

  - name: 生成变更摘要
    action: generate_changelog

  - name: 创建 PR
    action: github_create_pr
    params:
      template: "conventional-pr"
```

---

## 案例 4：文档生成

**场景**: 自动生成 API 文档

**解决方案**: 创建 `docs-gen` Skill

```yaml
name: docs-gen
description: 生成项目文档
trigger: /docs

steps:
  - name: 分析代码
    action: scan_code
    params:
      patterns: ["*.js", "*.ts"]

  - name: 生成 JSDoc
    action: generate_jsdoc

  - name: 构建文档
    action: build_docs
    params:
      format: markdown
```
