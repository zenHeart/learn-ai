# 自定义 Skill

## Skill 开发实战

### YAML 配置结构

```yaml
name: my-custom-skill
description: 这是一个自定义技能
trigger: /my-skill

steps:
  - name: 分析代码
    action: read_code
    params:
      files: "*.js"

  - name: 生成文档
    action: generate_docs
    params:
      format: markdown

  - name: 保存结果
    action: write_file
    params:
      path: "docs/README.md"
```

---

# 触发方式

<v-clicks>

1. **命令触发**: `/skill-name`
2. **自动触发**: 匹配特定模式时自动执行
3. **手动调用**: 在对话中显式调用

</v-clicks>

---

# 配置位置

* 项目级: `./.claude/skills/`
* 用户级: `~/.claude/skills/`

---

# 开发最佳实践

<v-clicks>

* **单一职责**: 每个 Skill 只做一件事
* **清晰描述**: 使用明确的名称和描述
* **错误处理**: 添加适当的错误处理
* **文档注释**: 说明输入参数和输出格式
* **版本控制**: 使用语义化版本号

</v-clicks>

---

# 示例：代码审查 Skill

```yaml
name: code-review
description: 自动化代码审查
trigger: /review

steps:
  - name: 获取变更
    action: git_diff

  - name: 分析代码
    action: analyze_code
    params:
      focus: ["security", "performance", "style"]

  - name: 生成报告
    action: format_report
    params:
      format: markdown
```
