# 1. 溯源：Skill 到底是什么？

> **本质上，Skill 就是大模型的 Tool Use (Function Calling) 的封装，再加上特定的上下文规则约束。**

::left::

<div class="mt-4">

### 💡 演进路线：从硬编码到 AI 编排

**过去 (硬编码工作流)**：
- 比如 ESLint 或 CI/CD 脚本。
- 只有 If-Else，遇到一点异常（格式不对）直接抛错退出。
- “人工定义每一步怎么走”。

**现在 (Skill / Tool Use)**：
- 给 AI 提供一把“锤子”（Tool）和一本“说明书”（Description/Schema）。
- AI 根据当前遇到的问题，**自适应地决定**什么时候用锤子，怎么用。

</div>

::right::

<div class="mt-4 bg-slate-50 dark:bg-slate-800/50 p-4 rounded-xl border border-slate-200">

### 对比：Skill 与 MCP/Agent

| 概念 | 底层原理 | 核心职责 |
|------|---------|---------|
| **MCP** | 协议/接口 | 提供**“武器库”** (让 AI 能访问数据库/外部API) |
| **Skill** | Tool Use | 提供**“SOP / 说明书”** (教 AI 怎么用武器组合打出连招) |
| **Agent** | 循环推理 | **“总指挥”** (调用不同的 Skill 和 MCP 解决复杂问题) |

</div>

# 本章小结：Skill 的本质

<v-clicks>

1. **不是魔法**：底层就是 OpenAI/Anthropic 原生的 Function Calling 能力。
2. **能力升维**：从传统 CI/CD 的“死板脚本”进化为“懂变通的 AI 流水线”。
3. **沉淀资产**：不管是 Claude 的 `Skill` 还是 Cursor 的 `.cursorrules`，本质上都是将团队的“潜规则”变成大模型的“思想钢印”。

</v-clicks>
