# 什么是 Harness？最小三件套

> Harness = **Tool + Memory + Loop**。三者缺一，AI 就无法持续稳定运行。

<div class="grid grid-cols-3 gap-6 mt-8">

<div class="p-4 rounded-lg" style="background: #eff6ff80; border: 1px solid #bfdbfe;">

### 🛠 Tool

让 AI 能"动手做事"

- 读 / 写文件
- 执行命令
- 调用 API

</div>

<div class="p-4 rounded-lg" style="background: #f0fdf480; border: 1px solid #bbf7d0;">

### 🗒 Memory

让 AI 能"跨会话记忆"

- Session：单次对话
- Project：仓库规范
- User：个人偏好

</div>

<div class="p-4 rounded-lg" style="background: #fffbeb80; border: 1px solid #fde68a;">

### 🔁 Loop

让 AI 能"自我纠错"

- 执行 → 观测 → 修正
- 不依赖人类确认每一步

</div>

</div>

<v-clicks>

- **没有 Tool**：AI 只能输出文字，无法修改代码
- **没有 Memory**：每次对话都从零开始
- **没有 Loop**：一次答错就放弃，不会自我修复

</v-clicks>

# Tool Calling：AI 申请，宿主执行

> AI 输出 JSON 描述"调用什么工具、参数是什么"，宿主执行后回传结果。

```ts {1-13|15-17|18-19|20-22|all}
// ① 宿主发送：用户消息 + 可用工具清单
const response = await client.messages.create({
  model: "claude-sonnet-4-5",
  messages: [{ role: "user", content: "/tmp 下有几个 .log 文件？" }],
  tools: [{
    name: "list_files",
    description: "列出指定目录下的文件",
    input_schema: {
      type: "object",
      properties: { path: { type: "string" } },
      required: ["path"]
    }
  }]
});

// ② 模型返回 tool_use（不是最终答案）
//    { type: "tool_use", name: "list_files", input: { path: "/tmp" } }
// ③ 宿主执行 list_files("/tmp") → ["a.log", "b.log", "c.log"]
//    回传 tool_result 给模型
// ④ 模型基于观测结果生成最终回答
```

<v-click>

**核心原则**：AI 是"裁判 + 教练"，工具执行权始终在宿主。安全边界由你掌控。

</v-click>

# Tool 设计原则：命名、描述、约束

> Tool 描述是给 AI 的 Prompt——含糊的工具，AI 无法正确调用。

::left::

<div class="col-bad">

### ❌ 坏 Tool

```ts
{
  name: "do_stuff",
  description: "处理数据",
  input_schema: {
    type: "object",
    properties: { data: { type: "string" } }
  }
}
```

- 名字模糊，AI 无法判断调用时机
- 描述缺使用场景
- 参数无约束，错误信息无意义

</div>

::right::

<div class="col-good">

### ✅ 好 Tool

```ts
{
  name: "search_pull_requests",
  description:
    "在仓库中按关键字搜索 PR。" +
    "适用：找重构合并记录、查 Bug 修复提交。" +
    "不适用：搜代码（请用 search_code）。",
  input_schema: {
    type: "object",
    properties: {
      query: { type: "string", minLength: 2 },
      state: { enum: ["open", "closed", "all"] }
    },
    required: ["query"]
  }
}
```

- 动词起头，名字即意图
- 描述含适用 / 不适用边界
- 参数约束明确，错误自解释

</div>

# Memory：跨会话持久化

> AI 无持久记忆——Memory 是它的外部存储，按生命周期分三层。

<div class="grid grid-cols-3 gap-4 mt-6">

<div class="p-4 rounded-lg" style="background: #f0f5ff; border: 1px solid #c7d2fe;">

### 📝 Session Memory

**单次对话内**

- 上下文窗口
- 多轮对话历史
- Token 耗尽即丢失

</div>

<div class="p-4 rounded-lg" style="background: #f0fdf4; border: 1px solid #bbf7d0;">

### 📂 Project Memory

**整个仓库共享**

- `AGENTS.md` / `CLAUDE.md`
- `.cursor/rules/*.mdc`
- 团队规范、架构决策

</div>

<div class="p-4 rounded-lg" style="background: #fff7ed; border: 1px solid #fed7aa;">

### 👤 User Memory

**跨项目跟随用户**

- 用户偏好（语言、风格）
- 个人 Skill / Command
- 历史学习记录

</div>

</div>

<v-click>

**关键约束**：<span v-mark.underline.red="1">**Agent 看不到的等于不存在**</span>。Slack 讨论、Google Docs、本地未提交配置，对 AI 都是黑洞。

</v-click>

# Ralph Loop：AI 自我纠错的闭环

::left::

```
人类输入需求 ──▶ ① 新会话（新鲜上下文）
                       │
                       ▼
                 ② 读 AGENTS.md（地图）
                       │
                       ▼
                 ③ 执行任务
                    （写 / 调试 / 文档）
                       │
                       ▼
                 ④ 自审 + Agent 评审
                       │
                       ▼
                 ⑤ 开 PR → CI 通过即合并
                       │
                       ▼
人类介入  ◀────  ⑥ 清理 → 下一轮
（仅边界）
```

::right::

### 三个关键约束

<v-clicks>

- **新鲜上下文**：每任务新开会话，避免历史污染
- **反压控制**：CI 失败 → Agent 自动修复，不阻塞其他任务
- **临时计划**：执行计划提交仓库，归档可追溯

</v-clicks>

<v-click>

### 落地：nn-client-all 查 Bug

1. 用户报告 "登录失败"
2. Agent 读 `AGENTS.md` → 定位 `src/auth/`
3. 跑测试 → 拿到 stack trace
4. 修代码 + 加用例 → 开 PR
5. CI 通过 → 自动合并

</v-click>

# Harness 反模式 & 本章小结

::left::

<div class="col-bad">

### ❌ 三大反模式

- **AGENTS.md 当百科全书**：1000 行 → 应 ~100 行地图
- **规范只靠文档**：README 写一遍 → 应 lint 强制
- **架构决策在 Slack**：Agent 黑洞 → 应转 `docs/design-docs/`

</div>

::right::

<div class="col-good">

### ✅ 三大原则

- **仓库即记录**：Agent 看不到的等于不存在
- **机械化强制**：lint 规则 > 文档约定
- **熵管理**：定期清理 Agent，债务如高息贷款

</div>

# 章节小结

<v-clicks>

✅ Harness 是 AI 持续稳定工作的**最小工程化环境**

✅ 三件套：**Tool（动手）+ Memory（记忆）+ Loop（自纠错）**，三者缺一为零

✅ **仓库即记录系统**：未提交的规范对 AI 不存在

✅ **机械化强制**：lint 规则 > 文档约定，CI 强制校验

</v-clicks>

<v-click>

> 本讲核心：从"AI 助手"升级为"AI 协作者"，需要完整的 Harness 支撑。

</v-click>
