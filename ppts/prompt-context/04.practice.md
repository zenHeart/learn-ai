# Prompt 框架

| 组成部分 | 功能 | 是否必须 |
|---|---|---|
| **指令 (Instruction)** | 希望模型执行的具体任务 | 必须 |
| **上下文 (Context)** | 补充信息，引导语言模型更好地响应 | 选填 |
| **输入数据 (Input Data)** | 需要模型处理的内容 | 选填 |
| **输出指示 (Output Indicator)** | 需要模型输出的类型或格式 | 选填 |

---

# Zero-Shot Prompting (零样本)

只包含指令，不提供任何示例

**示例：**
```
把下面这句话翻译成英文：今天天气真好
```

---

# Few-Shot Prompting (少样本)

<v-clicks>

包含指令，**示例输入和输出**，用来帮助模型学习回答模式

**示例：**
```
把以下中文翻译成英文：
例1：你好 -> Hello
例2：再见 -> Goodbye
请翻译：今天天气真好 ->
```

**注意：确保输入示例的质量，避免垃圾进垃圾出**

</v-clicks>

---

# Chain-of-Thought Prompting (CoT)

<v-clicks>

通过一系列思维提示辅助模型完成任务

**核心技巧：**
> "让我们一步一步地解决这个问题，以确保我们有正确的答案"

**效果：**
- 提升推理能力
- 减少错误率

</v-clicks>

---

# 组合技

<v-clicks>

* Zero-shot + CoT

# 核心原则

<v-clicks>

**1. 精确明确**
- 增加示例
- 拆解任务
- 增加引导词（代码生成）
- To Do And Not To Do（告诉 AI 做什么）
- 增加任务或角色
- 使用符号分隔内容
- 通过格式词调整输出

**2. 小心甄别**
- 模型可能返回错误结果
- 不要完全依赖模型
- 通过逐步调整 Prompt 获取最佳答案

</v-clicks>
