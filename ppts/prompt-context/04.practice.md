# 场景一：关键实践总结

<v-clicks>

**1. @file 引用替代文字描述**
- 直接引用 `@BaseTable.vue`，而不是"参考我们的表格组件风格"
- 真实代码 >> 语言描述

**2. .cursorrules 自动注入规范**
- 无需手动告诉 AI"用 Composition API"，规范文件已经注入 Context
- 团队每个人问出来的结果风格一致

**3. 明确数据结构（TypeScript Interface）**
- 在 Input 部分给出 Interface，AI 不会自造字段
- 减少"幻觉字段"的出现概率

**4. Output 部分要求测试骨架**
- 让 AI 同时生成测试文件骨架，强制 TDD 意识

</v-clicks>

# 场景二：最小有效 Context 切取策略

**正确做法：像外科手术一样精准切取**

<v-clicks>

**Step 1：定位核心调用链路**
- 看堆栈，找最近的业务代码帧（不是 node_modules）
- 通常只有 3-5 个关键函数

**Step 2：切取最小代码集合**
- 只复制报错的函数 + 它直接调用的函数
- 不要复制整个文件，只要相关的方法

**Step 3：结构化错误信息**
- 错误类型 + 完整堆栈
- 触发条件（什么情况下出现）
- 已经尝试过的方案

</v-clicks>

  await new Promise(r => setTimeout(r, 1000))

// pool.ts:82-95（调用方）

}
</code>

<question>

# 场景二：精准 Context 的 AI 输出

<v-click>

**AI 分析（直指根因）：**

```
根因分析：
这是一个典型的 TOCTOU（Time-of-Check-Time-of-Use）竞态条件。

1. handleDisconnect 先从 pool 删除 connection（line 84）
2. 之后调用 reconnect，reconnect 从 connections.get() 获取
3. 但 connections map 和 pool 是两个独立数据结构，删 pool 不删 connections
4. 重连期间如果再次触发 disconnect，ws 已经是 null

修复方案（不破坏现有接口）：

async reconnect(clientId: string) {
  const ws = this.connections.get(clientId)
  if (!ws) {                               // ✅ null 检查
    console.warn(`[WS] No connection for ${clientId}, skip reconnect`)
    return
  }
  await new Promise(r => setTimeout(r, 1000))
  if (this.connections.has(clientId)) {    // ✅ 二次确认，防止等待期间被删除
    ws.send(JSON.stringify({ type: 'ping' }))
  }
}
```

</v-click>

# 场景二：关键实践总结

<v-clicks>

**1. 堆栈 → 最小代码路径**
- 只提取 crash 相关的 3-5 个函数
- 保留行号（用于定位上下文）

**2. 错误信息结构化**
- 错误类型、触发条件、已尝试方案 → 分块清晰
- 用 XML 标签区分代码、错误、问题

**3. 明确问题边界**
- "类型安全且不引入 breaking change" → 约束 AI 的解决方向
- 没有约束 → AI 可能给出重构整个模块的"彻底方案"

**4. 不要问"有什么问题"**
- 要问"分析这个竞态条件的根因"→ 精确定位，而不是开放式扫描

</v-clicks>

# 两个场景的共同结论

<v-clicks>

**Context 的质量 = AI 输出的上限**

| 因素 | 低质量 Context | 高质量 Context |
|------|--------------|---------------|
| 信息量 | 越多越好（堆文件） | 最小有效集（精准切取） |
| 结构 | 自然语言描述 | XML/Markdown 结构化 |
| 规范 | 每次手动说明 | .cursorrules 自动注入 |
| 示例 | 无示例 | @引用现有代码作为 Few-Shot |
| 问题 | 开放式（"有什么问题？"） | 聚焦式（"分析这个竞态条件的根因"） |

</v-clicks>

<v-click>

> **最终结论**：Context Engineering 不是让你写更多，而是让你写得更准。  
> 精准 > 全面，结构 > 堆砌，约束 > 自由。

</v-click>
