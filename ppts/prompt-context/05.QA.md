# Q1：模型变笨了，是怎么回事？

<v-clicks>

**可能原因**：

1. **模型版本漂移**：厂商更新了模型，行为可能变化（不要用"latest"）
2. **Context 污染**：对话历史稀释了 System Prompt 的权重
3. **窗口截断**：Token 快满时，早期关键上下文被挤出
4. **厂商节流**：高流量时段模型响应质量下降

</v-clicks>

<v-click>

**应对策略**：

| 策略 | 场景 | 做法 |
|------|------|------|
| 新开会话 | 任务切换时 | 关键约束写入 CLAUDE.md，新会话自动加载 |
| 摘要压缩 | 长对话中途 | 让 AI 总结已达成的共识，清除历史重开 |
| 固定版本 | 关键任务 | 锁定模型版本，不随"latest"自动升级 |

</v-click>

# Q2：Prompt 写得好，真的是竞争力吗？

<v-clicks>

**是的**，但不是你想的那种。

</v-clicks>

<v-click>

**不是竞争力**的 Prompt：
- "请用中文回答"（模型原生支持）
- "请一步步思考"（已是标配）

**是竞争力**的 Prompt：
- 深度垂直领域的结构化模板（JSON 输出格式、边界约束）
- 与团队工程系统深度集成的规范（.cursorrules + lint + CI）
- 经过 AB 测试验证的 Few-Shot 示例库

</v-click>

<v-click>

> **核心**：Prompt 本身不值钱，**Prompt + Context + Harness 的系统化整合**才是护城河。

</v-click>

# Q3：Harness 和 Vibe Coding 有什么区别？

<v-clicks>

**Vibe Coding**：自然语言驱动 AI 完成单个任务，适合探索、原型阶段

**Harness 工程**：为 AI 建立可持续的工程环境，适合生产、团队协作

</v-clicks>

<v-click>

```
Vibe Coding：
用户："做个登录页面" → AI 全程完成 → 用户验收

Harness 工程：
用户："做个登录页面" → AI 读 AGENTS.md → 读 BaseComponent.vue
→ 按规范生成 → CI lint + 测试 → 自动合并
```

</v-click>

<v-click>

> **关系**：Harness 是 Vibe Coding 在团队、生产场景的工程化升级。

</v-click>

# Q4：把公司代码喂给 AI，安全吗？

<v-clicks>

**四层防护体系**：

</v-clicks>

<v-click>

| 层级 | 做法 |
|------|------|
| 文件级隔离 | `.cursorignore` 排除证书、密钥、核心算法目录 |
| 流水线拦截 | Git Hooks + `git-secrets` 强阻断敏感数据提交 |
| 意图标定 | `CLAUDE.md` 声明红线（如：绝不读取 /configs/prod） |
| 私有化部署 | 高密级场景用断网环境 + 本地模型（Ollama + DeepSeek-R1） |

</v-click>

# 下一步：如何开始

<v-clicks>

**今天下班前可以做的三件事**：

1. **写一个 .claudeprompt**：选择你最常用场景（代码审查 / 组件生成 / Bug 分析），把 System Prompt 提取到 `.cursorrules` 或 `CLAUDE.md`，纳入 Git 管理

2. **配置一层全局约束**：在项目根目录写入团队最核心的 3-5 条规范（禁止 any / API 走 src/api/ 等），让每次新开会话自动加载

3. **调一个 Tool**：如果你用 Cursor / Claude Code，试着写一个自定义 Tool（搜索项目文档 / 查 Git 历史），感受 Tool Calling 的机制
