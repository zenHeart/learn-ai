三层工程的真实疑惑深度解答

# Q1：Prompt 写完就完了？还是需要长期维护？

<v-clicks>

**短答**：Prompt 是活的，需要像代码一样迭代维护。

**为什么会失效：**
- 模型升级（GPT-4 → GPT-4o → o1），行为和默认风格变化
- 业务需求变化，原有约束不再准确
- 发现新的边界场景，原有 Prompt 没有覆盖

</v-clicks>

<v-click>

**工程化建议：**

| 实践 | 做法 |
|------|------|
| 版本管理 | 把核心 Prompt 放进 Git 仓库（如 `.cursorrules`、`CLAUDE.md`） |
| 回归测试 | 关键场景固定测试用例，每次模型升级后跑一遍 |
| 评估指标 | 为输出质量打分（1-5），追踪 Prompt 迭代效果 |
| 团队共享 | Prompt 是团队资产，不是个人秘密 |

</v-click>

# Q2：Context 越多越好吗？为什么加了信息反而更差？

<v-clicks>

**真相**：Context 越多，不等于越好。存在一个**精准阈值**。

**Lost in the Middle 效应**（Stanford, 2023）：
- 研究发现：模型对 Context **开头和结尾** 的信息注意力最高
- 塞在中间的信息，容易被忽视甚至"丢失"
- 20 个文档里找答案，文档放第 1 位和放第 10 位，准确率差距 **超过 20%**

</v-clicks>

<v-click>

**实践结论：**

```
❌ 把整个代码库塞进 Context  → 推理瘫痪，答非所问
✅ 精准切取最小有效 Context  → AI 聚焦，答案准确

关键信息 → 放开头或结尾
无关信息 → 坚决删掉，不要"以防万一"
```

</v-click>

# Q3：模型"变笨"了是怎么回事？

<v-clicks>

**现象**：长对话后，AI 的回答越来越差、越来越离题。

**根因**：不是模型变笨了，是 **Context 被填满了**。

- 每次对话，历史消息也占 Token
- 窗口快满时，早期的关键上下文被截断或权重降低
- 模型开始"忘记"你在第 3 轮说的架构约束

</v-clicks>

<v-click>

**三种应对策略：**

| 策略 | 场景 | 做法 |
|------|------|------|
| **新开会话** | 任务切换时 | 把关键约束放进 CLAUDE.md，新会话自动加载 |
| **摘要压缩** | 长对话中途 | 让 AI 总结已达成的共识，清除历史，粘贴摘要重开 |
| **滑动窗口** | 持续迭代时 | 只保留最近 N 轮，核心规范靠 System Prompt 固化 |

</v-click>

# Q4：把公司代码上下文喂给 AI，安全吗？

<v-clicks>

**正确姿势**：分级管控，不是一刀切禁止，也不是毫无防护。

</v-clicks>

<v-click>

**四层防护体系：**

</v-click>

<v-clicks>

🔒 **文件级隔离**：`.cursorignore` / `.gitignore` 排除证书、密钥、核心算法目录

🪝 **流水线拦截**：Git Hooks + `git-secrets` 强阻断敏感数据提交

📋 **意图标定**：在 `CLAUDE.md` 声明红线  
```
# 安全规则
- 绝不读取 /configs/prod 目录
- 绝不将线上用户真实数据作为示例输入
```

🏢 **私有化部署**：涉密程度高时，用断网环境 + 本地模型（Ollama + DeepSeek-R1）

</v-clicks>

# Q6：Harness Engineering 和 Vibe Coding 有什么本质区别？

<v-clicks>

**Vibe Coding** — 单人、单次任务的 AI 编程协作
- 你和 AI 对话，AI 帮你写某个功能
- 关注点：Prompt 写得好不好、Context 给得准不准
- 结果：这次 PR 质量高

**Harness Engineering** — 团队、持续交付的 AI 工程体系
- 你设计环境，AI 自主执行、PR、评审、修复
- 关注点：仓库结构、机械化约束、Agent 可读性、熵管理
- 结果：**整个系统自动运行，人类只在关键节点介入**

</v-clicks>

<v-click>

```
Vibe Coding   →  AI 是你的结对编程伙伴（单次任务）
Context 工程  →  AI 是你的高效执行者（稳定输出）
Harness 工程  →  AI 是你团队的一员（持续自主交付）
```

</v-click>

<v-click>

> Harness 不是替代 Vibe Coding，而是 Vibe Coding 规模化的工程基础设施。

</v-click>
