# 什么是 PE 工程

**提示工程（Prompt Engineering）**，就是探讨如何设计出最佳提示词，来指导语言模型，帮助我们高效完成某项任务。

---

# PE 工程有什么作用

<v-clicks>

* **提高输出质量** - 优化的提示工程可以帮助语言模型生成更准确、完整和相关的回答或结果。通过明确的指令和上下文框架，提示工程可以减少模型的歧义性和错误推理。
* **控制模型行为** - 通过调整提示词，引导模型遵循特定的指导原则、执行特定的任务或遵循特定的约束

</v-clicks>

---

# 核心原理：Prompt 进阶

## Zero-Shot Prompting (零样本)

只包含指令，不提供任何示例

**示例：**
```
把下面这句话翻译成英文：今天天气真好
```

---

## Few-Shot Prompting (少样本)

包含指令，**示例输入和输出**，用来帮助模型学习回答模式

**示例：**
```
把以下中文翻译成英文：
例1：你好 -> Hello
例2：再见 -> Goodbye
请翻译：今天天气真好 ->
```

**注意：确保输入示例的质量，避免垃圾进垃圾出**

---

## Chain-of-Thought Prompting (CoT)

通过一系列思维提示辅助模型完成任务

**核心技巧：**
> "让我们一步一步地解决这个问题，以确保我们有正确的答案"

**效果：**
- 提升推理能力
- 减少错误率

---

# 核心原理：Context 运作机制

## 注意力机制 (Attention)

<v-clicks>

LLM 使用 **注意力机制（Attention）** 来处理 Context

- 模型无法"记住"所有 Context
- 越靠近结尾的内容权重越高
- 重要信息应放在适当位置

</v-clicks>

---

## Token 上限与大海捞针

<v-clicks>

**为什么有限制？**
- 计算资源限制
- 成本控制
- 注意力衰减

**主流模型 Context 长度：**
- GPT-3.5: ~4K tokens
- GPT-4: ~8K-128K tokens
- Claude: ~100K-200K tokens

**Token** 是模型处理的基本单位，一般为 **4 个字符** 或 **0.75 个单词**

</v-clicks>

[Token 计算工具](https://platform.openai.com/tokenizer)

</v-click>

# Context 注入技巧

<v-clicks>

**1. 明确结构**
```
背景信息：
- 用户画像：...
- 使用场景：...
- 约束条件：...
```

**2. 重要信息前置**
- 把关键要求放在 Prompt 开头

**3. 分块输入**
- 长文本分段输入
- 使用分隔符清晰区分

</v-clicks>

---

# System Prompt 工程

<v-clicks>

System Prompt 是最高效的 Context 注入方式

**最佳实践：**
- 设定清晰的角色
- 明确输出格式
- 定义边界条件

**示例：**
```
你是一位专业的技术文档工程师...
请用 Markdown 格式输出...
避免使用...
```

</v-clicks>

---

# 对话历史管理

<v-clicks>

**问题**：长对话会超出 Context 限制

**解决方案：**

1. **摘要法** - 定期总结对话要点
2. **滑动窗口** - 只保留最近 N 轮
3. **选择性保留** - 提取关键信息

</v-clicks>

---

# RAG 与外部知识

<v-clicks>

**RAG (Retrieval-Augmented Generation)**

1. 用户提问
2. 检索相关知识
3. 将知识注入 Context
4. 生成回答

**优势：**
- 突破模型知识截止日期
- 减少幻觉
- 提供准确的专业信息

</v-clicks>

---

# Context 优化原则

<v-clicks>

1. **相关性** - 只提供与任务相关的信息
2. **清晰性** - 使用清晰的结构和分隔符
3. **简洁性** - 避免冗余和无关内容
4. **层次性** - 重要信息优先级高

</v-clicks>

---

# 常见 Context 问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 模型忘记之前内容 | Context 超出限制 | 摘要/分段 |
| 输出前后不一致 | 历史信息冲突 | 明确最新指令 |
| 幻觉回答 | 缺乏准确知识 | RAG 补充 |
| 答非所问 | Context 不清晰 | 优化结构 |
