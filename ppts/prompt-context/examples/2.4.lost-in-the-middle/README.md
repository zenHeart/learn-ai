# Lost in the Middle - 上下文关键信息遗忘

## 研究背景

Liu et al. 2023 在论文《Lost in the Middle: How Language Models Use Long Contexts》中提出了一个重要发现：

> **LLM 对上下文开头和结尾的信息回忆效果最好，而对中间位置的信息回忆效果最差。**

## 核心发现

### Attention U 型曲线

当关键信息位于上下文的不同位置时，LLM 的召回率呈现明显的 U 型分布：

```
        高
        ↑
       ╱ ╲
      ╱   ╲
     ╱     ╲
    ╱       ╲
   ╱         ╲
  ╱           ╲
 ─┴─────────────→
  开头  中间  结尾
```

### 关键实验

论文中设计了一个"50 条事实"的实验：

```
事实 1: Sam 在周五买了苹果
事实 2: 会议定在下午 3 点
...
事实 25: 会议在 305 房间 (关键信息)
...
事实 50: 下周二是公司团建日
```

**结果**：当"会议在 305 房间"被埋在中间位置时，模型的准确率显著下降。

### 为什么重要？

在实际应用中，这意味着一旦关键信息被"淹没"在长上下文的中间，LLM 可能会：
- 忽略该信息
- 给出错误答案
- 混淆不同位置的信息

## 解决方案

### 1. 关键信息前置或后置

将最重要的信息放在上下文的开头或结尾：

```
❌ 不好：[背景信息 1][背景信息 2]...[关键信息]...[背景信息 N]
✅ 好：[关键信息][其他上下文]
```

### 2. 重复关键信息

在多个位置重复关键点，确保被正确召回：

```
会议安排：会议在 305 房间
[大段背景信息...]
提醒：再次强调，会议在 305 房间举行
```

### 3. 使用明确的标记

用清晰的标记突出关键信息：

```
<关键>会议地点：305 房间</关键>
```

## 运行示例

```bash
node main.js
```

查看 `main.js` 获取模拟 Attention U 型曲线的代码。
