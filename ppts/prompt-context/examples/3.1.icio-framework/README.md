# ICIO 框架：结构化提示词的四大基石

## 什么是 ICIO 框架？

ICIO 框架是一种结构化提示词设计模式，将提示词拆分为四个核心组成部分：

| 组成部分 | 含义 | 作用 |
|---------|------|------|
| **Instruction** | 指令 | 明确告诉 AI 需要执行什么任务 |
| **Context** | 上下文 | 提供背景信息，帮助 AI 理解场景 |
| **Input** | 输入 | 待处理的数据或素材 |
| **Output** | 输出 | 指定期望的返回格式或结构 |

## 为什么使用 ICIO？

未结构化的提示词往往导致：
- AI 理解偏差，输出不符合预期
- 每次都需要重复解释背景
- 输出格式不一致，难以解析

ICIO 框架的优势：
- **清晰明确**：AI 能准确理解任务目标
- **复用性强**：Context 可复用，只需替换 Input
- **输出可控**：Output 规范确保格式统一

## 实战示例：从邮件中提取会议时间

### 场景

你收到了多封邮件，需要提取其中的会议时间安排。

### 未结构化的提示词

```
帮我从这些邮件里提取会议时间。
```

**问题**：没有明确提取什么格式、时间 timezone、如果有多个会议怎么办。

### 使用 ICIO 结构

**Instruction（指令）**
```
从邮件内容中提取所有会议时间安排，包括会议主题、时间和地点。
```

**Context（上下文）**
```
这些邮件来自你的日历助手，邮件中提到的会议时间均为北京时间（BJT）。
如果有多个会议，请按时间顺序排列。
```

**Input（输入）**
```
邮件1：
主题：项目评审会
正文：下周三下午3点在会议室A有项目评审会，欢迎参加。

邮件2：
主题：技术方案讨论
正文：周五上午10:30进行一次技术方案讨论，地点改到线上会议。
```

**Output（输出）**
```
请以 JSON 格式返回：
{
  "meetings": [
    {
      "topic": "会议主题",
      "datetime": "ISO 8601 格式时间",
      "location": "地点"
    }
  ]
}
```

### 对比效果

| 维度 | 无结构提示词 | ICIO 结构化 |
|------|------------|-------------|
| 主题提取 | 可能遗漏 | 明确提取 |
| 时间格式 | 随意文字 | ISO 8601 |
| 多个会议 | 格式不一 | 统一 JSON |
| 时区说明 | 缺失 | 明确标注 |

## 代码演示

运行 `main.js` 查看 ICIO 框架的实际效果：

```bash
node main.js
```

脚本会对比结构化与非结构化提示词的输出差异。
