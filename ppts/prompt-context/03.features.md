# Prompt + Context 组合

<v-clicks>

**1+1 > 2 的秘诀**

- Prompt 告诉模型 **"做什么"**
- Context 告诉模型 **"用什么做"**
- 两者结合才能精准控制输出

</v-clicks>

---

# 组合工作流

```
用户输入 → [Context 增强] → [Prompt 组装] → LLM → [输出优化] → 最终结果
                ↓                   ↓
            检索知识            结构化指令
            历史摘要            示例说明
            角色设定            格式要求
```

---

# 案例：智能客服

**Context:**
- 用户画像
- 产品信息
- 历史对话

**Prompt:**
```
你是一位专业客服，请根据以下信息回答用户问题。
用户问题：{question}
产品信息：{product_info}
```

# 案例：代码审查

```
审查标准：
```

**Prompt:**
```
请扮演资深工程师，对以下代码进行审查...
```

# 实战：构建你的工作流

<v-clicks>

**Step 1: 明确任务目标**
- 输入是什么？
- 输出是什么？

**Step 2: 准备 Context**
- 需要哪些背景信息？
- 如何获取和整理？

**Step 3: 设计 Prompt**
- 指令是否清晰？
- 格式要求明确？

**Step 4: 测试与优化**
- 效果如何？
- 哪里需要调整？

</v-clicks>

---

# 优化迭代

<v-clicks>

1. **先跑通** - 用最简单的 Prompt + Context 组合
2. **找问题** - 分析输出与预期的差距
3. **针对性优化**
   - Prompt 问题 → 优化指令
   - Context 问题 → 补充/精简信息
4. **验证效果** - 确认优化有效

</v-clicks>

---

# 最佳实践总结

<v-clicks>

**Prompt 层面：**
- 指令明确具体
- 使用框架结构
- 适当few-shot示例

**Context 层面：**
- 相关且最新
- 结构化呈现
- 注意长度控制

**组合层面：**
- Context 为 Prompt 服务
- 保持一致性
- 持续迭代优化

</v-clicks>
