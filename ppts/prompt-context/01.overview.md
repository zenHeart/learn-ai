# 从 Vibe Coding 出发，撞上三堵墙

<v-clicks>

上一讲，你学会了和 AI 协作编程。然而真实工作中，你一定遇到过这些时刻——

**场景 1：遗忘**
> 昨天告诉 AI 用 TypeScript strict、用 Pinia 管状态……
> 今天新开会话，它写出一堆 `any`，还用了 Vuex。

**场景 2：幻觉**
> 生成的代码看起来非常合理，跑起来才发现——
> 它调用了一个根本不存在的 API，且信心满满。

**场景 3：无序**
> 同一个问题，今天问和明天问，结果完全不同；
> 团队每个人问出来的代码风格也完全不同。

</v-clicks>

<v-click>

> 三堵墙的解药：**遗忘** → Memory/Context（会话记忆）；**幻觉** → Context 工程（精准投喂）；**无序** → Harness 工程（团队对齐）。

</v-click>

---

# 根本约束：AI 只能看见 Context Window

> AI 的能力，取决于它能看到什么。

<v-clicks>

**Context Window 三个事实**：
- 主流最大 **1M Token**，大部分 **200K Token**
- 新开会话 = **空上下文**，一切从零开始
- 注意力机制对**位置敏感**——开头与结尾权重高，中间易被忽略

</v-clicks>

<v-click>

```
┌──────────────────────────────────────────────────┐
│             Context Window（上下文窗口）          │
│                                                   │
│  ┌────────────┐  ┌────────────┐  ┌──────────┐  │
│  │  System    │  │  对话历史   │  │ 当前输入  │  │
│  │  Prompt    │  │（滚动窗口） │  │          │  │
│  └────────────┘  └────────────┘  └──────────┘  │
│                                                   │
│  AI 每次回复，只能看到窗口内的内容                 │
└──────────────────────────────────────────────────┘
```

</v-click>

<v-click>

> 那么，**如何在有限窗口里做对的事**？接下来三章逐一回答。

</v-click>

# 三者关系：包含 + 装配 + 调度

> 三层不是并列，而是**嵌套**——Prompt ⊂ Context ⊂ Harness。

<v-click>

```
┌──────────────────────────────────────────────────────────┐
│  Harness（运行时环境）                                     │
│  Tool 调用 · Memory 持久化 · Loop 自我纠错                 │
│                                                          │
│    ┌────────────────────────────────────────────────┐   │
│    │  Context（本次窗口实际内容）                    │   │
│    │  System + Memory 注入 + 历史 + Tool 结果       │   │
│    │                                                │   │
│    │    ┌──────────────────────────────────────┐   │   │
│    │    │  Prompt（你写的当前输入字符串）       │   │   │
│    │    └──────────────────────────────────────┘   │   │
│    │                                                │   │
│    └────────────────────────────────────────────────┘   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

</v-click>

<v-clicks>

- **你能直接控制的**：Prompt（当前输入）
- **模型实际看见的**：Context（窗口内全部 Token）
- **决定 Context 怎么装的**：Harness（每轮 Loop 重新组装一次）

</v-clicks>

<v-click>

> 数据流：**你写 Prompt → Harness 组装 Context → 模型推理 → Loop 反馈**——任何一环断裂，整体失效。

</v-click>

# 演进时间线：Prompt → Context → Harness

> AI Coding 工程化的三波浪潮——你今天踩的坑，正是这条演进线推过来的。

<v-click>

| 时期 | 关键能力 | 学术/产品里程碑 | 主战场 |
|------|---------|---------------|-------|
| **2020–2022** | Few-Shot · CoT | Brown 2020 / Wei 2022 / ChatGPT | **Prompt** 工程化 |
| **2023–2024** | 长上下文 · RAG | GPT-4 128K / Liu 2023 / Claude 200K | **Context** 工程化 |
| **2024–2025** | Tool Use · Agent | MCP / Cursor / Claude Code | **Harness** 工程化 |
| **2026（今天）** | 三层融合 | Extended Thinking · Subagent · Skill | **Vibe Coding 全栈** |

</v-click>

<v-click>

```
2020 ──────── 2023 ──────── 2025 ──────── 2026
 │             │             │             │
 Prompt ─────▶
              Context ─────▶
                           Harness ─────▶
                                       三层融合
```

</v-click>

<v-click>

> **每一波都是上一波的进化**：Prompt 解决"问得对"，Context 解决"喂得准"，Harness 解决"跑得久"。

</v-click>
