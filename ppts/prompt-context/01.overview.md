# 三个让人崩溃的时刻

<v-clicks>

**场景 1：遗忘**
> 昨天告诉 AI 用 TypeScript strict mode、用 Pinia 管状态……  
> 今天新开会话，它写出了一堆 `any`，还用了 Vuex

**场景 2：幻觉**
> AI 生成的代码看起来非常合理，跑起来才发现——  
> 它调用了一个根本不存在的 API，且信心满满

**场景 3：无序**
> 同一个问题，今天问和明天问，结果完全不同  
> 团队每个人问出来的代码风格也完全不同

</v-clicks>

---

# 根因只有一个

你在和 AI <span v-mark.underline.red="1">**对话**</span>，但你需要的是对 AI 的 <span v-mark.circle.blue="2">**工程化控制**</span>。

<v-click at="3">

```
❌ 随机对话     →  每次结果不同，无法复用，无法团队共享
✅ 工程化控制   →  可重复、可预测、可迭代的 AI 工作流
```

</v-click>

<v-click at="4">

**三层工程栈，就是这把钥匙：Prompt · Context · Harness。**

</v-click>

---

# 这一讲的目标

<span v-mark.underline.yellow="1">掌握 Prompt · Context · Harness 三层工程方法，构建可持续演化的 AI 工作流。</span>

# 先建立一个直觉

**Context Window（上下文窗口）** 是理解一切的基础。

<v-click>

```
┌─────────────────────────────────────────────────────┐
│                    Context Window                    │
│                                                     │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────┐ │
│  │ System Prompt│  │ 对话历史      │  │ 当前输入  │ │
│  │ (全局规则)   │  │ (短期记忆)   │  │ (你的问题)│ │
│  └──────────────┘  └──────────────┘  └───────────┘ │
│                                                     │
│  AI 每次回复，只能看到这个窗口里的内容               │
└─────────────────────────────────────────────────────┘
```

</v-click>

<v-clicks>

- AI **没有持久记忆**，新会话=白板
- 窗口有 **Token 限制**，塞不下整个代码库
- **你放进去什么，决定了它给你什么**

</v-clicks>

# Prompt vs Context：两个不同的问题

> Prompt 决定任务方向，<span v-mark.circle.red="3">Context 决定输出质量</span>。大多数人只关注 Prompt，忽视了 Context 才是关键。

::left::

<v-click>

### Prompt（提示词）
<span v-mark.underline.yellow="1">告诉 AI **做什么**</span>

- "帮我写一个 Vue3 Table 组件"
- "分析这段代码的性能问题"
- "翻译这段文字"

</v-click>

::right::

<v-click>

### Context（上下文）
<span v-mark.underline.blue="2">告诉 AI **在什么情况下做**</span>

- 这个项目用 TypeScript strict mode
- 组件风格参考 `src/components/BaseButton.vue`
- 不能用 Class 组件，只用 Composition API

</v-click>

# 章节小结

<v-clicks>

✅ AI 不是聊天工具——它是一个**有上下文限制的概率推理引擎**

✅ 遗忘/幻觉/无序，根源都是 **Prompt 不精准 + Context 管理失控 + 环境没有工程化**

✅ 解法：三层工程栈缺一不可

</v-clicks>

<v-click>

```
Prompt 工程  →  告诉 AI 做什么（如何问）
Context 工程 →  告诉 AI 在什么情况下做（喂什么）
Harness 工程 →  设计 AI 可持续工作的环境（如何跑）
```

</v-click>
