# 为什么需要 Prompt + Context

<v-clicks>

**痛点 1**: 同样的模型，不同的结果
- 同一个问题，不同的 Prompt 给出天差地别的答案

**痛点 2**: 上下文丢失
- 长对话后模型"忘记"之前的内容

**痛点 3**: 能力被低估
- 很多人没有充分发挥大语言模型的潜力

</v-clicks>

---

# 什么是 Prompt 工程

**提示工程（Prompt Engineering）**，就是探讨如何设计出最佳提示词，来指导语言模型，帮助我们高效完成某项任务。

<v-clicks>

* **提高输出质量** - 优化的提示词可以帮助语言模型生成更准确、完整和相关的回答
* **控制模型行为** - 通过调整提示词，引导模型遵循特定的指导原则、执行特定任务

</v-clicks>

---

# 什么是 Context 工程

<v-clicks>

**Context（上下文）** 是指：
- 对话历史
- 用户提供的背景信息
- 系统指令
- 外部知识库

**Context 工程** 就是有意识地组织和管理这些上下文信息，让模型更好地理解任务。

</v-clicks>

# 预备知识：核心概念

* **Token** - 模型分析时的基本单位，一般为 4 个字符，大约 0.75 个单词
* **Temperature** - 改变输出内容的随机性，越高每次答案越多样
* **Top_p** - 取值 0-1，越高内容越随机，和 temperature 概念相似
* **Max_tokens** - 最大输出 token 限制

<v-click>

# 预备知识：AI 发展脉络

* **AI** (人工智能)
  * **ANI** (弱人工智能) - 人脸识别、自动驾驶、AlphaGo、ChatGPT
  * **AGI** (强人工智能) - 做任何人类可以做的事情

* **NLP** (自然语言处理)
  * **NLU** - 理解自然语言
  * **NLG** - 自然语言生成

# 预备知识：机器学习基础

**机器学习** - 计算机通过大量样本的学习，完成特定的目标

<v-clicks>

* **训练集** - 用于训练模型的数据
* **特征** - 数据的可区分属性
* **建模** - 学习的过程
* **模型** - 训练后形成的规律

</v-clicks>

训练方法分类：
* 监督学习
* 非监督学习
* 强化学习
