# 三个让人崩溃的时刻

<v-clicks>

**场景 1：遗忘**
> 昨天告诉 AI 用 TypeScript strict mode、用 Pinia 管状态……
> 今天新开会话，它写出了一堆 `any`，还用了 Vuex

**场景 2：幻觉**
> AI 生成的代码看起来非常合理，跑起来才发现——
> 它调用了一个根本不存在的 API，且信心满满

**场景 3：无序**
> 同一个问题，今天问和明天问，结果完全不同
> 团队每个人问出来的代码风格也完全不同

</v-clicks>

<v-click at="+1">

> **平均一次会话烧 8 万 Token** ——大部分是浪费在它翻找上下文上的

</v-click>

---

# 根因：Context Window 的根本限制

> AI 的能力取决于它能看到什么上下文。

<v-clicks>

**Context Window 限制**：
- Claude 3.7 Sonnet：200K tokens
- 新开会话 = 空上下文，一切从零开始
- Token 有限，塞不整个代码库

</v-clicks>

<v-click>

```
┌─────────────────────────────────────────────────┐
│              Context Window（上下文窗口）           │
│                                                  │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────┐ │
│  │ System      │  │ 对话历史    │  │ 当前输入 │ │
│  │ Prompt     │  │（滚动窗口） │  │          │ │
│  └─────────────┘  └─────────────┘  └──────────┘ │
│                                                  │
│  AI 每次回复，只能看到窗口内的内容                │
└─────────────────────────────────────────────────┘
```

</v-click>

<v-click>

> **遗忘 / 幻觉 / 无序**，根源都在于 Context 管理失控。

</v-click>
